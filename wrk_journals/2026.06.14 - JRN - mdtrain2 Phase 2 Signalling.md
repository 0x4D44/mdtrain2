### Design — DRAFT  ✓ (+ critical team-lead ruling)
3 angled HLDs → 3 judges → **Draft B (risk-first) won 3-0, unanimously.**
- **CRITICAL RULING (P2-RA) — team lead overrode the synthesis agent.** The judges (3-0) caught a load-bearing bug: `tickSafety` seeds `const reasons = new Set(s.penaltyReasons)` (controls.ts:258) — carries ALL prior reasons forward — and the release deletes only "DSD" (:278-281). So a folded "AWS"/"TPWS" reason persists forever → **un-releasable penalty latch.** I verified this directly against the code. The V1 synthesis agent then *reverted* to the simpler-but-broken A/C view ("the reason leaves on the shared ack edge"), which is FALSE against the carry-forward seed — V1's C2 release oracle would fail.
- **Decision:** adopt Draft B's fix — scope the carry-forward seed to **DSD-only** (`new Set(); if (s.penaltyReasons.has("DSD")) reasons.add("DSD")`), re-deriving AWS/TPWS fresh from the fold each tick; `tickAws` owns its AWS/TPWS latch lifecycle and stops returning the reason when cleared. This is a one-line MERGE-half edit at :258; the RELEASE machine (277-281), `resolveInputs`, `reduceControls` stay unchanged. Rationale: it is the only contract-true way to make AWS/TPWS releasable; "don't change the latch" = "don't change the release machine," which is honoured.
- Correcting V1 to bake in this ruling before the review loop (don't ship a known-broken keystone into review).

### Design — REVIEW round 1  (4 critical + 4 major verified; producing V2)
Round 1 (4 lenses, 2-of-3 verify) on the corrected V1. Resolutions (team lead):
- **F1 (crit) — one-tick release lag.** The AWS BRAKE-release condition was byte-identical to the latch release (both ack-at-stand), making "survive-ack" unobservable + self-contradictory. Fix: `tickAws` re-asserts `brakeReason` on the ack-at-stand tick (fold re-adds it → C1/C3 survive) and applies CLEAR/null on the NEXT tick (returns [] → DSD-scoped seed drops it → C2/C2-PIN releasable). Update steps 5-6, the C-oracles.
- **F2 (crit) — MODEL DECISION: red is the platform STARTER at the departure end.** "Signal-before-platform, red-until-served" made every stop a guaranteed SPAD/deadlock (you'd have to SPAD the red to reach the platform that clears it). New model: each station's protecting RED is the **starter just PAST the stop board**, red until that station is served; AWS magnet ~180 m on approach; signals further back show yellow/double-yellow by lookback. Approaching → AWS warning → ack → stop & serve → starter clears → depart. A SPAD/TSS only happens if you blow the red starter without stopping. Decouples "reach platform" from "pass red". (Mirrors legacy mdtrain.)
- **F3 (crit)** A1 oracle precondition fixed to match `aspectAt` (self-held tested first).
- **F4 (crit)** crossing detection generalised: `pointsCrossedFwd(points, sPrev, sNow)` over three position arrays (posts, posts−AWS_OFFSET, posts−OSS_OFFSET) — the single signal-chainage query couldn't see magnets/loops.
- **F5/F6/F8/F10 (major)** pass `dir` (from `lastDir`) into `tickAws`; guard WARNING/CLEAR on `phase!==BRAKE` (latched brake outranks fresh warning); drop duplicated `aspect` threading (compute in `buildHudView`); fix determinism-oracle fixtures (held-RED signal, served constant). F7/F9 not confirmed (1/3, 0/3) → documented, not blocking.

### Design — REVIEW round 2  (1 critical verified; verify step heavily rate-limited → lead-judged the rest; producing V3)
NOTE: round-2 verify was hit by API rate-limiting (19/… skeptic votes failed). Only F1 fully verified; I judged the remaining findings myself from their claims (read against code).
- **F1 (crit, verified) — STARTER_OFFSET geometry.** `STARTER_OFFSET=60 < platformHalf (90–100)`, so a legal stop in the upper 30–40 m of a platform sweeps the RED starter post **while still moving** (before serving) → spurious SPAD/TPWS. The V2 "SPAD-free by construction" keystone breaks. Fix: `STARTER_OFFSET=120` (> max platformHalf 100 + FRONT_OFFSET 2), re-derive the route table; add an **overshoot oracle** (rest front in (post, board+platformHalf] ⇒ NO spad).
- **F8 (simplicity) — RECONCILIATION: drop the one-tick lag (reverses round-1 F1's `releaseArmed`).** Re-reasoned: the DSD-scoped seed (P2-RA) already gives releasability; the genuine "survive" case is **ack-WHILE-MOVING** (C4), not "survive a stand-ack while still demanding." AWS/TPWS legitimately clear *on* the ack-at-stand, so the lag served a non-requirement. **Keep the seed-scoping; drop `releaseArmed`/the lag; clear same-tick on ack-at-stand; reframe C-oracles** so "survive" = ack-while-moving (C4), "releasable" = ack-at-stand drops same tick (C2/C2-PIN pins that the DSD-scoped seed is required). Simpler and correct. (Round-1 F1's lag was an over-correction.)
- **Other (lead-judged real):** F2 buildHudView ripples to **8** O14 sites (not 4); F3 frame reorders so tickSafety/tickAws run AFTER step with prevChainage=pre-step chainage; F4 build G1/G2 determinism fixtures DIRECTLY (synthetic prev/now positions, not via clamped step()); F5 add B4 (BRAKE outranks a fresh magnet warning); F6 strengthen G2 (within-tick AWS-vs-TPWS precedence); F7 author 1–2 plain GREEN block signals (with magnets) so the sunflower can reset to BLACK. Minors F9–F12 folded where cheap.
