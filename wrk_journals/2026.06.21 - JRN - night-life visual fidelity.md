## Robust harness adopted ‚úÖ (commit 6066b97)
`playwright.config.ts` + `e2e/night.spec.ts` (dev-only `@playwright/test`, matched to the installed
`playwright@1.61.0` + cached chromium-1228). The runner owns the browser lifecycle ‚Üí **no more
orphaned chromium**. THREE root causes of the old flakiness, all fixed:
- **Lost key presses (the big one).** Under SwiftShader the rAF loop runs *far* slower than 60 fps,
  so a bare `press()` (keydown+keyup back-to-back) had its keyup `edges.delete` the code **before any
  frame read it** (main.ts drains+clears edges per frame). `tap()` now holds the key down across **‚â•2
  REAL rAF frames** (`page.evaluate(requestAnimationFrame‚Ä¶)`, frame-rate-independent) before release.
  This was the actual cause of `e`/`f` "not registering" ‚Äî proved with a code probe.
- **Screenshot stall.** `page.screenshot()` hung after "fonts loaded" on the perpetual WebGL canvas;
  a `clip`-ed viewport capture (30 s budget) is reliable. SwiftShader composite/readback is ~20‚Äì26 s
  per shot (bloom under software GL) ‚Üí test timeout raised to 120 s. Slow but rock-solid.
- Data-driven `SHOTS` at each set-piece (`?s=` seed), focus `#app`, cycle to wet-night, optional
  drive / look-around, capture, **assert zero console/page errors** (the real gate).
