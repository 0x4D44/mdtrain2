## Robust harness adopted ✅ (commit 6066b97)
`playwright.config.ts` + `e2e/night.spec.ts` (dev-only `@playwright/test`, matched to the installed
`playwright@1.61.0` + cached chromium-1228). The runner owns the browser lifecycle → **no more
orphaned chromium**. THREE root causes of the old flakiness, all fixed:
- **Lost key presses (the big one).** Under SwiftShader the rAF loop runs *far* slower than 60 fps,
  so a bare `press()` (keydown+keyup back-to-back) had its keyup `edges.delete` the code **before any
  frame read it** (main.ts drains+clears edges per frame). `tap()` now holds the key down across **≥2
  REAL rAF frames** (`page.evaluate(requestAnimationFrame…)`, frame-rate-independent) before release.
  This was the actual cause of `e`/`f` "not registering" — proved with a code probe.
- **Screenshot stall.** `page.screenshot()` hung after "fonts loaded" on the perpetual WebGL canvas;
  a `clip`-ed viewport capture (30 s budget) is reliable. SwiftShader composite/readback is ~20–26 s
  per shot (bloom under software GL) → test timeout raised to 120 s. Slow but rock-solid.
- Data-driven `SHOTS` at each set-piece (`?s=` seed), focus `#app`, cycle to wet-night, optional
  drive / look-around, capture, **assert zero console/page errors** (the real gate).

## Slice 3 — celestial layer + trackside markers ✅ implemented + tested (targeted-visual ✓)
- **Pure (`environment.ts`):** new `nightFactor` ∈ [0,1] on `EnvironmentParams` (day 0 · dusk 0.2 ·
  night 1), time-only, strictly increasing — the single clean scalar that fades the celestial layer.
  Unit-tested (O13 ordering + endpoints; extended O12 finite/pure). **322 tests green** (was 320).
- **`scene.ts`:** moon disc (`MeshBasic`, fog:false) + additive halo sprite (`makeRadialGlowTexture`
  in textures.ts) + 520-star `Points` (seeded → stable shots), all in one `celestial` group that
  **follows the eye** (sky-box) and is **faded by nightFactor**. Built ONCE; only the group origin +
  opacities change per frame ⇒ allocates nothing on env change, **cannot leak** (AC-D), simpler than
  the proof's rebuild-on-env. **Deviation from DL2:** moon at a FIXED hero direction over the line
  (`MOON_DIR = (-0.2,0.4,0.9)`, screen-right pane, clear of the cab A-pillar), NOT `sunDir` — the env
  vector points high+behind, outside the ±137°/+40° look arc, so it'd never be seen. The proof also
  uses a fixed moonDir; day/night presence rides on nightFactor. Verified in `09-moon.png` (hero
  moon+halo) and `04-open-country.png` (stars + markers).
- **`scenery.ts`:** `buildMarkerLights` — one InstancedMesh of warm emissive glints (0xfff0d0) along
  the ballast edge, alternating sides every 24 m, `anchorY` on the ground, skipping viaduct/bore.
  **Emissive-only, ZERO PointLights** (DL4/R1 respected). Reads as a lit lineside corridor at night.
- `tsc` clean. Per-slice visual gate: full 9-shot suite green (zero console/page errors). Committed b3eeb40.
