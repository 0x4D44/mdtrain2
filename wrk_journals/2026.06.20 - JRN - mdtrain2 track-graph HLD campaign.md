## Phase 1 — UNDERSTAND
Launched `tg-understand` (3 readers: signalling/interlocking foundations; simulation/physics/multi-
train; prior-art/constraints/graph-foundations). Building the structured brief the drafts will use.
### UNDERSTAND synthesis (3 readers converge)
**Keystone:** the tested core extends by PARAMETERIZING THE ORIGIN, not rewriting.
- **Edge** = a `Route` restricted to one node-to-node span. ALL geometry math
  (placeOnCentreline/planarPoseAt/heightAt/headingAt/centerlineAt/cantAt/terrainHeight) is
  byte-identical on an Edge — the ONLY change is the integration origin (ψ0,x0,z0,h0) becomes an
  `EdgeFrame` instead of hard-wired ψ=0,(0,0),0 (centerline.ts:92-94). Single route = EdgeFrame
  identity (the special case).
- **TrackGraph** = nodes (points = shared EdgeFrames) + edges; a node hands exit-frame→entry-frame.
  Continuity oracle: entryFrame(child)==exitFrame(parent) ≤1e-6 (the two-arc test already proves it).
- **TrainPosition {edgeId,s,d}** replaces `SimState.chainage`. Physics `step()` UNCHANGED; a pure
  **`stepOnGraph`** wrapper carries the residual onto the next booked edge at s≥edge.length
  (reconcile orientation/sign at the junction — the most error-prone seam, needs its own oracle).
- **Path** = ordered edges = the booked route (player + each AI service); "scenario sets the points"
  ⇒ Path is DATA (no interlocking UI, no pathfinding).
- **Occupancy** = a DERIVED predicate from all TrainPositions, swapped into the `aspectAt` cascade in
  place of/alongside `served.has()`: source Set = served ∪ blocks-clear-ahead. Train in the block
  ahead ⇒ protecting signal RED ⇒ follower holds. "Delay the AI by occupying a block" = a richer Set
  source, NO new primitive, keeping the R→Y→YY→G cascade + its termination proof.
- **AI controller** = pure `aiInputs(...)` producing the SAME `SimInputs` as `resolveInputs` (a
  target-speed tracker: min of PSR / red-yellow braking curve v²/2a / booked path). Plugs in at the
  exact resolveInputs boundary. Player keeps resolveInputs; AI uses aiInputs; both feed step().
- **Multi-train tick** order-independent: occupancy from frame-N positions, then step all on the
  shared clamped dt (preserves "advance first, then signal"). Per-train (state,controls,safety,aws).
- **Render**: AI trains are NEW meshes via placeOnCentreline(edge,s,0)+heading (the existing lineside
  primitive). Player camera stays single-train.

**Hard constraints:** G3 determinism grep (no clock/RNG in src/sim) — new pure modules in src/sim;
O17 layering (sim must not import render); dt-slice independence for steady intervals (dt≤0.05);
the **latch release contract** (a persisting occupancy hold re-adds itself by failing a "source no
longer demands it" predicate WITHOUT touching the release machine — the documented seam); per-route
invariants (minCurveRadius≥250/O-RIBBON, bands on κ=0, cant-off-ceiling PSR) become PER-EDGE + a
graph-level oracle.

**Open decisions for the drafts:** (1) junction orientation/sign reconciliation oracle; (2)
stepOnGraph as a new layer over the unchanged step() [recommended]; (3) block granularity (whole-edge
vs sub-interval; new block-signal kind vs overlay on starters); (4) AI fidelity (PSR+braking only vs
+platform dwell/timetable); (5) tick tie-break for block contention; (6) node geometry (frame-handoff
vs turnout); (7) AI trains' signalling (full AWS machine vs reduced read-aspect-and-obey); (8)
TrainPosition.d (parallel track as d-offset on one edge vs distinct edges).

## Phase 3 — ADVERSARIAL REVIEW LOOP
Launched `tg-review` over V1 (mixed lenses + a standing DEVIL'S ADVOCATE; loop until a round yields

### Round 1 result (workflow w1r5sfzb0, 24 agents): 1 real critical (5 lenses + devil converged), V1 has a FATAL flaw
19 findings; 6 candidate-criticals; **5 confirmed** — but they are ONE finding restated by 5 lenses
(the coarse title-dedup didn't merge them):

**THE critical — the keystone is self-contradictory.** The "unchanged step()" end-clamp
(`simulation.ts:88-91`) sets `chainage=route.length; speed=0` INSIDE the substep loop the instant
`chainage>=length && speed>0`. So `stepOnGraph`'s carry guard `next.speed>0` is NEVER true at a
join, the residual is always 0, and every train **dead-stops at the end of its first edge** — S1/S2/
G-DIFF/J1/AI2 all unreachable. "Carry the residual over the byte-identical step()" cannot both hold.
Numerically verified by reviewers (train at s=399.5, v=20, dt=0.05 on a 400m edge → returns
{400, 0}). The devil's advocate nailed the meta-point: the "zero tested lines / byte-identical step()"
selling point is precisely the thing that has to give.

**Fix (read simulation.ts to ground it):** gate ONLY the forward end-clamp (line 88) behind an
additive `clampAtEnd = true` parameter. `route.length` is used nowhere else in step(), so this is
clean. Every existing caller uses the default ⇒ behaviorally byte-identical ⇒ all existing oracles
pass. `stepOnGraph` passes `clampAtEnd=false` on a non-terminal edge (chainage runs past length, speed
retained ⇒ real residual) and `true` on the terminal/buffer-stop edge (J3 survives). This is the
narrowest honest change; the reviewers explicitly endorsed it over the literal-unchanged framing.

**1 dismissed critical** (refuted 3/3): flat Set<string> can't encode the lower-id tie-break asymmetry
— moot, because the tie-break itself is being cut (below).

**Majors + minors → a simplification cluster I'm adopting (it REVERSES my V1 lower-id graft):**
- **Cut RESERVE_M look-ahead reservation + the lower-trainId tie-break.** They defend a same-frame
  double-entry race the testbed (dt≤0.05, edges≫step, staggered starts) never exercises — an on-ramp
  to nowhere. Make occupancy **purely physical** (a train occupies its current edge only). This
  dissolves the dismissed-critical, both reservation majors (withoutOwn self-footprint; cuttable
  RESERVE_M), and the scope-creep minor at once. MT1 (order-independence, the real hard constraint)
  survives trivially via the frame-N snapshot; merge same-frame double-entry becomes a documented
  deferred non-goal (avoided by authoring; add reservation then if ever needed).
- **Namespaced block tokens** (major: protects-station-vs-block-conflation) — block token = edge id;
  validateGraph asserts station-ids and edge-ids are disjoint, so the richer source Set never aliases
  the station-starter invariant. aspectAt stays verbatim.
- **Merge the three fenced 1-D modules into one `graph-sim.ts`** (minor) — graph-geom MUST stay
  separate for O17, but graph-step/ai/multitrain are all on the fenced side; one file holds the
  reactive-AI story. New sim modules: graph.ts (types) / graph-geom.ts (spatial) / graph-sim.ts (1-D)
  / testbed.ts (data) = 4, not 6. O17 fence count 5→7 (not the wrong "5→8").
- **O17 transitive leak** (minor) — add `graph-geom` to the fenced-modules denylist so a fenced module
  importing it fails the build; enforces the honesty the split claims.
- **validateGraph edge-length invariant** (minor) — assert every edge.length > maxSpeed·MAX_DT (comfort
  margin) so one-join-per-tick is machine-enforced; drop V1's hand-wavy "defensive single re-entry".
- **G-DIFF** (minor) — narrow the independence claim (carry/distance half is the independent part vs a
  carry-free flat route; pose half is O5b composition); use level/constant-grade edges so the residual
  integrates identically on both sides ⇒ exact to 1e-9.
- **Testbed E_loop** (minor) — an out-and-back loop sharing both merge endpoints is necessarily LONGER
  than the straight; drop the inconsistent "~400m"; note grade-continuity at joins.
- **advance/signal label** (minor) — reword: occupancy sampled once from frame-N positions, then all
  trains advance; the single pre-move snapshot is what makes the tick permutation-invariant.
