### 3.1 `sliceRoute` — guarded sub-route extraction (DR/D-IMPL-2; pure; throws on a hazardous cut)

```ts
export function sliceRoute(route: Route, sFrom: number, sTo: number): Route;
```
Algorithm: assert `0 ≤ sFrom < sTo ≤ route.length`. **Cleanliness guard — throw if any holds:**
`curvatureAt(route, sFrom) !== 0` or `curvatureAt(route, sTo) !== 0` (cut not on a κ=0 straight); any
station with `|chainage − cut| < platformHalf` for cut ∈ {sFrom,sTo} (platform straddle); any signal
post or its derived reach `[chainage−AWS_MAGNET_OFFSET, chainage]` straddling a cut, OR a retained
signal whose `protects` station is not retained (orphaned cascade); any viaduct/tunnel band
`[center−halfLen, center+halfLen]` partially in [sFrom,sTo] (truncated band). Otherwise build:
- `length = sTo − sFrom`.
- each Segment band: keep segments overlapping [sFrom,sTo], clip `from=max(from,sFrom)`,
  `to=min(to,sTo)`, then subtract `sFrom` from from/to. (lookup is half-open + end-clamping ⇒ clean.)
- stations/signals: keep those with chainage ∈ [sFrom,sTo); subtract sFrom from chainage.
- viaducts/tunnels: keep bands wholly inside; subtract sFrom from center. Drop `terrainSeed` (or keep).
Unit-tested: a clean slice round-trips (gradeAt/curvatureAt/speedLimitAt of the slice at s == original
at s+sFrom); a hazardous cut throws.

```ts
//   over nextSignalAhead(edge.route, r.state.chainage, 1) & aspectAt(edge.route, n.i, aspectSource(EMPTY,…)).
```
Note vs HLD §3.7: `spec` lives ON the record (no external `spec(r)` map) — cleaner, same behaviour.
Imports: simulation (step/SimState/SimInputs/createInitialState), physics (TrainSpec), route
(Aspect/aspectAt/nextSignalAhead/speedLimitAt), graph. **No centerline, no graph-geom.**

> **AMENDMENT (Phase-4 review C1) — supersedes the HLD §3.7 "AI base served = EMPTY".** The HLD's
> glue note assumed the AI's base served set is empty, which is correct ONLY for the synthetic testbed
> (whose edges carry no station starters). The **live KINGSGATE decomposition carries the real station
> starters** (Ashcombe @2120, Wealdham @5920, …), and `aspectAt` is RED iff `!source.has(sig.protects)`,
> so an empty-served AI reads every station starter as RED and freezes at the first one. **Fix:** a
> `TrainRecord` carries `served` = the station names on its booked path (`stationsOnPath`), used as
> `baseServed`; the AI's source is then `served ∪ clear-block-tokens`. A station starter (protects a
> station NAME, which is in `served`) clears; a block signal (protects an EDGE ID) still REDs when its
> block is occupied. This **cannot** alias because **OCC-NS** asserts station-names ∩ edge-ids = ∅. The
> synthetic testbed keeps `served = ∅` (unchanged). The player record's `served` is unused (it is driven
> by `playerInputs`, not `aiInputs`). Pinned by the new live-KINGSGATE end-to-end oracle.
