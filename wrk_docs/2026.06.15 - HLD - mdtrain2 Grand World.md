The world is a flat green plane the rails lie *on*. This overhaul makes it a country the line **threads through** — cuttings that cut, embankments that carry, a river valley spanned by a **viaduct**, a headland bored by a **tunnel** — on a longer, seven-chapter **Grand Route**, lit with ACES tone-mapping, PBR materials, one real sun and a real sky. The governing instinct: **the world is geometry, and geometry is math.** Every metre of track, every terrain height, every prop's place, every "is this span a viaduct?" decision is a *pure function of route data in curvilinear coordinates*, computed and checked on Node with no GPU. Three.js is a thin, named, screenshot-verified projector that draws what the pure layer computed.

## 1. Scope & non-goals

### In scope
1. **A pure curvilinear spatial core** under `src/sim`, three small pure modules:
   - **`centerline.ts`** (path only): `centerlineAt(route,s) → Frame{x,y,z, tangent, up, heading, cant}`; `heightAt` (= ∫grade, analytic); `headingAt`; `planarPoseAt(route,s) → {x,z,heading}`; `placeOnCentreline(route,s,d) → {x,y,z, heading}` — the single placement primitive for every lineside object **and** set-piece structure (returns the *mathematical* track heading; Three facing offsets are applied render-side, D21).
   - **`terrain.ts`** (imports `centerline`): the **two-surface** model — `formationHeight(route,s)` (the rail/deck top) vs `terrainHeight(route,s,d)` (the ground), which coincide near the track normally but **diverge** in viaduct/tunnel bands; plus `reliefAt`/predicates (on **macro** relief), `anchorY`, `clampEye`.
   - **`camera.ts`** (imports both): `cabAttitudeTarget(cant,grade,scale) → {pitch,roll}` (pure clamp; eased render-side) and `eyePose(route,s,eyeD,eyeHeight) → {position, heading}` (placeOnCentreline + `clampEye`). *(Restored as its own module — folding it into `centerline.ts` created an ESM cycle, D1''.)*
2. **A real living landscape:** a build-once, seam-welded, sectioned **terrain-ground ribbon**; a **track ribbon** (rails/sleepers/ballast) always at `formationHeight` along the centreline; a **multi-arch viaduct** (deck at formation, piers to the natural valley floor) and a **tunnel** (hill ground + a bore cutout + portal/bore shell), both placed via `placeOnCentreline`; **sea/river** planes; every prop via `placeOnCentreline`+`anchorY`. The flat ground, cone "hills" and faked `addBank` cuttings are **deleted**.
3. **A push toward realism, free-first, tier-gated:** `ACESFilmicToneMapping` set on the renderer (applied once on either the direct-to-canvas or the `OutputPass` composer path — D17); a **color-space-correct** pipeline (D23); a **palette re-tune** of the existing emissives (D14); **procedural canvas textures + normal maps** (no committed image assets); a **gradient sky dome** (`fog:false`); a **tiny 16×16 procedural equirect env map** on `scene.environment` for cheap IBL ambient + wet-rail sheen (D5'); and — **tier-gated** — **one** eye-tracking shadow sun and an **optional, async-loaded** bloom pass. Everything scales down via `quality.ts`; nothing breaks if it's all off.
4. **A grand multi-chapter route:** `KINGSGATE_SEAHAVEN`, ~**14 km**, **7 chapters**, optional `viaducts[]`/`tunnels[]`/`terrainSeed`. `WESTFORD_EASTBANK` **retained as a test fixture** (physics/signalling only; never terrain-rendered).

### Non-goals
No 2-D track plan (D2). No physics change (1-D `gradeAt`; never imports the 3-D path — O17). No world→chainage inversion (curvilinear `(s,d)`). No chunked/streaming terrain (build-once sectioned ribbon + far-horizon band + fog — D8). No C1 easements (D11). No full scene-cube PMREM, water sim, modelled tunnel interior, god-rays, day/night cloud sim. No new npm dependency. No env vars, no feature flags. No migration/back-compat except keeping `WESTFORD_EASTBANK` as a fixture.

## 2. The design

### 2.0 Architecture in one breath
```
route data (1-D bands + optional viaducts/tunnels/terrainSeed)
   │
   ├─ physics.ts ── gradeAt(route,s)                                   [UNCHANGED 1-D path]
   │
   ├─ centerline.ts (PRIME SEAM) ── centerlineAt/heightAt/headingAt/planarPoseAt/placeOnCentreline
   │       └─ terrain.ts (imports centerline) ── formationHeight; terrainHeight(s,d); reliefAt; macro
   │             predicates viaductSpanAt/tunnelBoreAt; anchorY; clampEye
   └─ camera.ts (imports centerline + terrain) ── eyePose(route,s,eyeD,eyeHeight); cabAttitudeTarget

environment.ts ── …+ exposure, sunDir, sunColorPbr, bloomStrength
quality.ts     ── …+ shadowsEnabled, bloomEnabled, ribbonHalfWidth, terrainSegLen, terrainSubdiv,
                   attitudeScale; lodForDistance(d); ribbonVertexCount(tier,len)

render/* (IMPURE): read the pure layer; apply Three facing offsets + the cab-attitude ease; place & draw. No spatial math.
```
Everything left of `render/*` is pure, under `src/sim`, banned-token-free (integer-hash noise; trig allowed), oracle-tested. **G3 must be rewritten (not tweaked):** it globs **two** patterns — `src/sim/*.ts` **and** `src/render/quality.ts` — merged into one map (a single `import.meta.glob` can't reach both — completeness #5); it **strips line/block comments** from each source; and it matches a **word/call-boundary-anchored** regex (`\bDate\b`, `\bMath\.random\b`, `\bperformance\.now\b`, `\bset(Timeout|Interval)\b`) so neither comments nor identifiers like `updateDate` false-positive, with a **positive-control** assertion that the matcher does flag a known-bad sample (D6'/D26, O17).

### 2.1 `src/sim/centerline.ts` (NEW, PURE, TESTED) — the path
```ts
export interface Frame { x:number; y:number; z:number; tangent:Vec3; up:Vec3; heading:number; cant:number; }
export function centerlineAt(route: Route, s: number): Frame;
export function heightAt(route: Route, s: number): number;            // ∫₀ˢ grade (analytic ⇒ C0)
export function headingAt(route: Route, s: number): number;            // tangent yaw, rad (heading=0 ⇒ tangent=+Z)
export function planarPoseAt(route: Route, s: number): { x:number; z:number; heading:number };
export function placeOnCentreline(route: Route, s: number, d: number): { x:number; y:number; z:number; heading:number };
```
- **Height** = `Σ grade_i·(seg∩[0,s])`, clamped; piecewise-linear ⇒ C0 (O1–O3).
- **Planar pose:** heading `ψ(s)=ψ₀+∫κ`; per segment compute the LOCAL arc displacement and **rotate by the entry heading ψ₀** before adding to the running pose (`x += (cosψ₀−cos(ψ₀+θ))/κ`, `z += (sin(ψ₀+θ)−sinψ₀)/κ`, `θ=κL,R=1/κ`); straight advances `L` along `ψ₀`. ψ carries across breakpoints ⇒ G1 tangent (O4, O5, O5b). **`lateralOffsetAt` is removed** — the model is an accumulated `(x,z,heading)` pose, not a single offset (finding #8); use `planarPoseAt`.
- **`placeOnCentreline(s,d)`** returns world `(x,y,z)` = `planarPose + d·right` (`right=(cos heading, −sin heading)`) with `y = centerlineAt(s).y`, and the **mathematical `heading`** (heading=0 ⇒ +Z). **Three facing offsets are render-side (D21):** a camera uses `heading+π`; a mesh modelled front-+Z uses `heading`; a rear-facing signal lamp uses `heading+π`. (O5c pins position incl. the left/right **sign**; O5d pins heading semantics.)
- **cant** = `clamp(CANT_GAIN·|κ(s)|·v_ref(s)², CANT_MAX)`, signed; `v_ref=speedLimitAt(s)`; **pinned `CANT_GAIN=0.08`, `CANT_MAX=0.105 rad`** (O6).
- `Math.sin/cos/sqrt/atan` allowed; no clock. Honest `z(s) ≤ s` on curves; physics never reads it (R6).

### 2.1b `src/sim/camera.ts` (NEW, PURE, TESTED) — the eye + cab attitude
```ts
export function cabAttitudeTarget(cant: number, grade: number, scale: number): { pitch:number; roll:number };
export function eyePose(route: Route, s: number, eyeD: number, eyeHeight: number): { x:number; y:number; z:number; heading:number };
```
- `eyePose` = `placeOnCentreline(s, eyeD)` lifted by `eyeHeight`, with `y = clampEye(route, s, eyeD, place.y + eyeHeight)`. **Pinned `EYE_HEIGHT=1.9`, `EYE_D=−0.5`.**
- `cabAttitudeTarget(cant,grade,scale) = { pitch: clamp(scale·atan(grade), ±PITCH_MAX), roll: clamp(scale·cant, ±ROLL_MAX) }`. **Pinned `ROLL_MAX=0.070`, `PITCH_MAX=0.052`, `attitudeScale=0.35`(desktop)/`0`(coarse & reduced-motion).** *Defensive ceilings:* in the live path `cant≤CANT_MAX` and grades are gentle, so `scale·CANT_MAX≈0.037<ROLL_MAX`; O16 exercises the ceilings with synthetic over-clamp inputs.
- **One pitch path (finding #3):** the camera's pitch is the **eased cab pitch + look.pitch only** — *not* a separate raw `atan(grade)` term. The render layer eases the live `(pitch,roll)` toward `cabAttitudeTarget` via `approach()` each frame and sets `camera.rotation = (easedPitch + lookPitch, heading + π + lookYaw, easedRoll)` (YXZ); `cabMount` takes the same eased pitch/roll, `heading + π`, no look offset. *(Module split resolves the ESM cycle: `centerline` is path-only, `terrain` imports `centerline`, `camera` imports both — D1''.)*

### 2.2 `src/sim/terrain.ts` (NEW, PURE, TESTED) — the **two-surface** living land
The key correction (Codex CRIT1, D20): the **track/deck** and the **ground** are two surfaces.
```ts
export function formationHeight(route: Route, s: number): number;          // rail/deck TOP (+ ballast). The train runs here, ALWAYS.
export function macroGround(route: Route, s: number, d: number): number;    // authored macro landform (valley/hill plateaus), NO noise
export function naturalGround(route: Route, s: number, d: number): number;  // macroGround + bounded seeded value noise
export function terrainHeight(route: Route, s: number, d: number): number;  // the GROUND surface (see below)
export function macroReliefAt(route: Route, s: number): number;             // macroGround(s,0) − formation(s)  [NO noise]
export function bandMembership(route: Route, s: number): number;            // m(s) ∈ [0,1]: smooth viaduct/tunnel band weight (macro-only)
export function viaductSpanAt(route: Route, s: number): boolean;           // macroReliefAt < −VIADUCT_THRESH
export function tunnelBoreAt(route: Route, s: number): boolean;            // macroReliefAt >  TUNNEL_THRESH
export function boreCorridorAt(route: Route, s: number, d: number): boolean; // tunnelBoreAt(s) && |d| < BORE_HALF (render-omitted)
export function anchorY(route: Route, s: number, d: number, clearance: number): number; // terrainHeight + clearance
export function clampEye(route: Route, s: number, eyeD: number, eyeY: number): number;  // ≥ terrainHeight(s,eyeD)+MIN_EYE_CLEARANCE
```
**The ground surface — one s-continuous formula (D20 + D27):**
```
ballastShoulder(|d|) = BALLAST_DROP            (constant for |d| ≤ SHOULDER; ramps only in the shoulder beyond)
bedGround(s,d)  = formation(s) + ballastShoulder(|d|)                         // the clean rail bed
normGround(s,d) = lerp( bedGround(s,d), natural(s,d), w(d) ),  w(d)=smoothstep(0,BLEND_W,|d|−SHOULDER)  // cutting/embankment
m(s)            = bandMembership(s) ∈ [0,1]    // 0 outside, 1 inside a viaduct/tunnel band; smooth, ramps over SHOULDER_RAMP
terrainHeight(s,d) = lerp( normGround(s,d), natural(s,d), m(s) )              // C0-continuous in s AND d, everywhere
```
**Pinned** `SHOULDER=4 m`, `BLEND_W=80 m`, `BALLAST_DROP=0.4 m`, `VIADUCT_THRESH=18 m`, `TUNNEL_THRESH=12 m`, `SHOULDER_RAMP=120 m`, `BORE_HALF=6 m`.
- **`m(s)` (band membership, pure, macro-only — D27):** a smooth `smoothstep` of how far `|macroReliefAt(s)|` has crossed its threshold, ramping `0→1` over `SHOULDER_RAMP`. So `m=0` on normal spans (⇒ `terrainHeight=normGround`, the clean bed), `m=1` fully inside a band (⇒ `terrainHeight=natural`, the valley/hill), and a **smooth handoff across the band edge** — there is **no cliff**. `m` reads only the macro field (no noise), consistent with D24, so the same `SHOULDER_RAMP` that shapes the macro plateau also eases the near-track ground.
- **Normal spans:** near the track `terrainHeight≈formation+ballast` (cutting/embankment meet the bed); the track ribbon sits on it.
- **Viaduct spans:** as `m→1` the near-track ground eases **down to the natural valley** over `SHOULDER_RAMP`; the **track ribbon stays at `formationHeight`**, carried on a **separate viaduct deck + piers** (render-side §2.6h) down to the valley floor (`terrainHeight` under each pier). The masonry **abutment + wing-walls** at each mouth face the (now ramped, not cliff) embankment-to-deck transition.
- **Tunnel spans:** as `m→1` the near-track ground eases **up into the natural hill**; the **track ribbon stays at `formationHeight`** inside a **bore** — render-side, the ground ribbon's near-track corridor (`|d| < BORE_HALF`) is **omitted** and a **bore shell (a dark half-cylinder roof following `placeOnCentreline` over the whole band) + portals** are drawn so the line is visible through the headland.
- **O8 (continuous):** `m=0` (normal) ⇒ `|d|≤SHOULDER ⇒ terrainHeight==formation+BALLAST_DROP` (exact, ≤1e-9, since `ballastShoulder` is constant there); `m=1` (deep in a band) ⇒ `terrainHeight==natural`; **and O8-edge: `terrainHeight` is C0-continuous in `s` — `|terrainHeight(s⁻)−terrainHeight(s⁺)| ≤ NOISE_MAX` across every band edge** (the `m(s)` ramp guarantees no step). A pure `boreCorridorAt(s,d)=tunnelBoreAt(s) ∧ |d|<BORE_HALF` marks the render-omitted corridor (so the "visible bore" has a pure handle, even though hollowness itself is screenshot-gated — finding #6).

**Macro vs noise (Codex MAJ4, D24):** authored macro-relief is a **flat-bottomed plateau + `SHOULDER_RAMP` smoothstep shoulder** (pinned `SHOULDER_RAMP=120 m`): a trough of `valleyDepth` per `viaducts[].center±halfLen`, a `hillHeight` bump per `tunnels[]`. **Predicates threshold on `macroReliefAt` (macro only, no noise)**, and the **set-piece structures are placed from `route.viaducts[]`/`tunnels[]` data directly** (not by scanning a noisy field). The value noise is **bounded**: `|valueNoise| ≤ NOISE_MAX` (pinned `NOISE_MAX=6 m`), and `NOISE_MAX < VIADUCT_THRESH=18`/`TUNNEL_THRESH=12` with margin, so noise can **never flip a predicate** (O9 tests the bound). The noise still rolls the *ground* between set-pieces; it just can't conjure or erase a structure.

**`naturalGround`** = `macroGround` + a seeded integer-lattice value noise (mulberry-style hash on `terrainSeed + (⌊s/CELL⌋,⌊d/CELL⌋)`, bilinear, 2–3 octaves, amplitude-clamped to `NOISE_MAX`) — pure, banned-token-free. **`anchorY`** is a tested invariant (O7); **`clampEye`** floors the eye (O20, `MIN_EYE_CLEARANCE=0.5 m`).

### 2.5 `src/sim/route.ts` (CHG, data) — the Grand Route


### 2.6 Render adapters (IMPURE, screenshot-verified) — no spatial math
- **`src/render/terrain-mesh.ts` (NEW).** Build ONCE on a single unified `(s,d)` grid over **`s ∈ [−RIBBON_OVERHANG, length + RIBBON_OVERHANG]`** (pinned **`RIBBON_OVERHANG = 200 m`**, so the world doesn't end at the buffers and the eye at `s=0`/`s=length` sees ground ahead/behind), `d ∈ [−ribbonHalfWidth, +ribbonHalfWidth]`, Y = `terrainHeight(s,d)`. **Outside `[0,length]` the pure functions extrapolate along the entry/exit tangent** (`route.ts:lookup` already clamps grade/curvature to the end segments, so `heightAt`/`planarPoseAt`/`terrainHeight` continue straight off each end — completeness blocker #1; O-DOMAIN pins continuity across `s=0`/`s=length`). The `m(s)` blend makes the ground C0 across band edges (no tear). **Seam contract:** central-difference normals on the unified grid FIRST, then slice into `RIBBON_SECTIONS=ceil(length/SECTION_LEN)` (pinned `SECTION_LEN=900 m`) sections sharing boundary verts+normals identically (cull independently, no popping). **In tunnel bands omit the near-track corridor `boreCorridorAt(s,d)` (`|d|<BORE_HALF`)** and draw a **dark bore shell — a half-cylinder roof following `placeOnCentreline(s,0)` at deck height over the WHOLE band length** (not just the end portals, so the omitted corridor under the hill is never an open trench — finding #1). PBR + procedural textures; static **far-horizon band** behind; fog hides the edge. A separate **track ribbon** (rails/sleepers/ballast) runs along `placeOnCentreline(s,0)` at `formationHeight(s)` — always, including the viaduct deck and the bore floor. (Track-ribbon + sleeper instances are swept/instanced and **out of the O18 ground-ribbon budget** — finding #8.)
- **`src/render/scene.ts` (CHG).**
  - **(a) Camera/cab:** `eyePose` → `camera.position`; the impure layer eases `(pitch,roll)` toward `cabAttitudeTarget` and sets `camera.rotation.set(easedPitch + look.pitch, heading + π + look.yaw, easedRoll)` (YXZ — the **`+π` faces +Z**, Codex CRIT2). **`cabMount.position = eyePose.position` (the same eye point)** and `cabMount.rotation = (easedPitch, heading + π, easedRoll)` (no look offset) so the cab furniture stays locked to the eye on curves (completeness #7).
  - **(b) Lineside via `placeOnCentreline`:** **signal heads, stations, masts, fences, mileposts, AWS markers** positioned at `placeOnCentreline(chainage, sideOffset)`; **each mesh gets its Three facing offset** (front-+Z meshes use `heading`; signal lamps/boards use `heading+π` to face the approaching train). The **rails, sleepers and the contact wire** — today single full-length `BoxGeometry(…, route.length)` boxes that cannot bend — become **swept/segmented geometry along the spine** (a short box per `terrainSegLen`, or a tube), at `formationHeight` (completeness #8).
  - **(c) Rain** anchored to the eye `eyePose.position (x,y,z)` — **not the hard-coded `x=0`** of today's straight-line code; the slant follows the eye `heading`, not a fixed +Z (completeness #6).
  - **(d) Tone-mapping (D17):** `renderer.toneMapping=ACESFilmicToneMapping`, `renderer.outputColorSpace=SRGBColorSpace` set once. No-composer (mobile) path applies ACES on the canvas render. Composer path: `RenderPass(linear)→UnrealBloomPass(linear HDR)→OutputPass`, where `OutputPass` reads `renderer.toneMapping` and applies ACES+sRGB once. `toneMappingExposure=env.exposure`.
  - **(e) Sky + IBL (D23):** a gradient **sky dome** `material.fog=false`, zenith = render-local darkening of `env.skyColor`. ## 4. Acceptance criteria (numbered oracles — definition of done)

**Pure unit oracles (vitest, Node, no GPU):**
- **O1–O3** height: analytic integral ≤1e-9; differential vs `gradeAt` ≤1e-6 off breakpoints; C0 ≤1e-9.
- **O4** tangent G1 ≤1e-6 across every curvature breakpoint; `‖tangent‖≈1`.
- **O5** single-arc closed form (ψ₀=0); **O5b** two-arc compounded pose vs hand-computed (≤1e-6, catches rotate-by-ψ₀).
- **O5c** `placeOnCentreline(s,d)` exactly `|d|` from the spine, perpendicular — **including the left/right sign**: on a straight (heading=0), `d=−0.5 ⇒ x=−0.5` (catches mirrored `d`, finding #9).
- **O5d** heading semantics (PURE test in `camera.test.ts` — finding #4): straight ⇒ `heading=0` (tangent=+Z), `z(s)=s`; and the forward direction implied by `rotation.set(0, heading+π, 0)` under YXZ equals `(0,0,+1)` (heading=0 ⇒ camera looks +Z), proving the `+π` convention with no GPU.
- **O6** cant non-tautological (below-clamp equals the formula; over-clamp equals `CANT_MAX`; sign).
- **O7** anchor: `anchorY−terrainHeight===clearance`, `anchorY≥terrainHeight`.
- **O8 (continuous, two-surface)** `m(s)=0` (normal span) ⇒ `|d|≤SHOULDER ⇒ terrainHeight==formation+BALLAST_DROP` (≤1e-9, exact on curves; `ballastShoulder` is constant there — finding #3); `m(s)=1` (deep in a band — assert ∃ a sampled `s` per band where `m===1` exactly, **non-vacuity**) ⇒ `terrainHeight===naturalGround`; and **O8-edge: `|terrainHeight(s−ε,d)−terrainHeight(s+ε,d)| ≤ NOISE_MAX` across every band edge** for all sampled `d` (the `m(s)` ramp guarantees no cliff — findings #11/#12/#13). `bandMembership` is monotone `0→1` over `SHOULDER_RAMP`, `∈[0,1]`, and **macro-only: bit-identical for two different `terrainSeed`** (it never reads noise — completeness oracle-side).
- **O8b — track sits on its bed (no clip).** For all sampled `(s,d)` with `|d|≤SHOULDER` on `KINGSGATE_SEAHAVEN` (incl. inside both bands and the `m(s)` ramps), `formationHeight(s) ≥ terrainHeight(s,d) − ε` — the track ribbon is never below the ground (the "rails don't sink" property as a pure oracle, not just S6).

## 6. Decision log
*(D1–D19 carry from V3 unless noted.)*
- **D1''** **`camera.ts` restored as its own module** (reversing V2's D1' fold): folding the camera helpers into `centerline.ts` created an ESM cycle (`cameraFrame`→`clampEye` in `terrain`→`heightAt` in `centerline`). Split: `centerline` path-only, `terrain` imports `centerline`, `camera` imports both. *(Codex MAJ3.)*
- **D20 (CRITICAL fix) — two-surface terrain.** Separate `formationHeight` (the track/deck the train always runs on) from `terrainHeight` (the ground). They coincide near the track on normal spans, but in **viaduct** bands the ground = the natural valley (the deck is carried separately on piers) and in **tunnel** bands the ground = the hill (the bore is a render-side cutout + shell). A single heightfield could represent neither (it filled the valley / couldn't bore the hill). O8 is conditional. *(Codex CRIT1 — the headline external fix.)*
- **D21 (CRITICAL fix) — camera/Three yaw convention.** `placeOnCentreline`/`headingAt` return the *mathematical* heading (heading=0 ⇒ tangent=+Z). The render layer applies the Three facing offset: camera `rotation.y = heading + π + lookYaw` (Three cameras look down −Z, so +π faces +Z — matching today's `BASE_YAW=π`); front-+Z meshes use `heading`; rear-facing signal lamps use `heading+π`. V3 omitted the +π and would have pointed the camera down the line backwards. *(Codex CRIT2.)*
- **D22 — async bloom over a sync scene.** `createScene` stays synchronous; on `bloomEnabled` it lazy-`import()`s the composer and renders direct-to-canvas (ACES on the renderer) until ready, then switches. Mobile never starts the import (S5). *(Codex MAJ6.)*
- **D23 — color-space + env-map mapping pinned.** Albedo/emissive/env canvas textures `SRGBColorSpace`; normal/roughness data `NoColorSpace`; the env texture `mapping=EquirectangularReflectionMapping` (else PMREM won't treat it as an env map); rebuilt/disposed safely on env change. *(Codex MAJ5.)*
- **D24 — predicates on macro relief; bounded noise.** `viaductSpanAt`/`tunnelBoreAt` threshold on `macroReliefAt` (no noise); set-piece structures are placed from `route.viaducts[]`/`tunnels[]` data; `|valueNoise| ≤ NOISE_MAX (6 m) < THRESH` so noise can't flip a predicate (O9). Renders the set-pieces from data, not by scanning a noisy field. *(Codex MAJ4.)*
- **D25 — `O-RIBBON` scoped to terrain-rendered routes.** The `ribbonHalfWidth ≤ 0.5·minCurveRadius` invariant (and the 400 m floor) applies to `KINGSGATE_SEAHAVEN`; the retained `WESTFORD_EASTBANK` fixture (250 m curve, physics/signalling only, never terrain-rendered) is exempt. *(Codex MAJ7.)*
- **D26 — `lateralOffsetAt` removed; G3 comment-stripped + word-boundary anchored; O5c gains a sign assertion.** The path model is an accumulated `(x,z,heading)` pose (`planarPoseAt`), not a single offset; G3 strips comments AND anchors on word/call boundaries; O5c pins the left/right sign; O5d is a pure camera-yaw test. *(Codex minors #8/#9/#10 + confirmation #4/#5.)*
- **D27 (confirmation fix) — the two-surface ground is ONE s-continuous formula, not a boolean branch.** V4 hard-switched `terrainHeight` between a norm branch and a natural branch on the boolean predicate, producing a ~THRESH-tall **C0 cliff** in the ground at every viaduct/tunnel band edge (caught by 3 lenses). V5 blends them with a smooth, macro-only **band-membership weight `m(s)∈[0,1]`** ramping over `SHOULDER_RAMP`: `terrainHeight = lerp(normGround, natural, m(s))`. This makes the ground **C0-continuous everywhere** (O8-edge), is **simpler** (one formula, no branch), and the masonry **abutment/wing-walls (viaduct)** + **bore shell/portals (tunnel)** cover the deck-vs-ground gap. The earlier "O8 is conditional" framing (D20) is superseded: **O8 is continuous** and adds an edge-continuity assertion. *(Confirmation round, findings #1/#2/#3/#6/#7/#8/#11/#12/#13.)*
