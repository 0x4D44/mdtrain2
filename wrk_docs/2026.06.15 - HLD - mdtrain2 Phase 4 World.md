# HLD — mdtrain2 Phase 4: World (stations, scenery, the weather system)

**Status:** SIGNED-OFF · Date 2026-06-15 · **Approved by:** self-signed under delegated authority (Arthur granted "design+implement in one run" + "full autonomy, both")
**Provenance:** lead-authored directly against current HEAD (proportionality — a content/presentation phase with low design-risk; the one load-bearing seam, weather→adhesion, is carved pure & tested). Target `C:\language\mdtrain2` (Phase 0-3 merged, v0.4.0). `package.json` 0.4.0 → 0.5.0.

Phase 3 gave us a *fixed* wet-night cab over a schematic world (rails, sleepers, three signal heads, rain). Phase 4 makes the **world** read like a railway and turns the fixed wet-night into a **selectable environment system** whose weather drives **both** the look **and** the physics (rail adhesion). It adds **no** new train control and changes **no** sim-core *physics* — it adds one new pure sim module (`environment.ts`) and enriches the render world.

---

## 1. Scope & non-goals

### In scope
- **Environment system** (`src/sim/environment.ts`, NEW, PURE, TESTED): a small
  `Environment = { time, weather }` model and a pure `environmentParams(env)` that
  returns **physics** (`mu` adhesion) **and** visual params (sky/fog colour+range,
  ambient/moon intensity, rain intensity, rail wetness, wiper on/off). A pure
  `cycleEnvironment(env)` steps a small preset ring (wet-night is the default and
  first entry). This is the load-bearing testable seam: weather → μ → the *real*
  longitudinal physics already in `physics.ts`.
- **Station structures** (expand `src/render/scene.ts`): for each `route.stations`
  — a **platform** slab beside the running line, a **canopy**, an illuminated
  **name board**, and platform **lamps** (cheap emissive + a small point light).
  Built once from primitives; instanced where it pays.
- **Lineside scenery** (expand `src/render/scene.ts`): **OLE / catenary masts**
  marching along the route (it's an EMU — overhead line is signature), lineside
  **fencing**, schematic **distant buildings**, and **mileposts**. All instanced,
  no per-frame allocation.
- **Weather wired to look + physics**: `scene.ts` consumes the visual params each
  frame (background/fog, lights, rain opacity, rail sheen, wiper gate); `main.ts`
  feeds `environmentParams(env).mu` into `resolveInputs` (replacing the hardwired
  `ADHESION.wetNight`) and projects the visual params into the `RenderView`.
- **A demo affordance to switch environment** (one key) so the system is visible;
  the default stays the signature wet-night.

### Non-goals
- Photoreal scenery, textured/LOD building models, render-to-texture reflections,
  shadows. The world stays **schematic** (emissive + sheen + instanced primitives).
- Lengthening the line / new stations (scope discipline: the existing
  Westford→Eastbank segment, done richer — *not* longer).
- New train controls or gameplay; signalling/timetable/scoring changes; new deps.
- A full settings UI for environment (that's a one-key demo cycle here; richer
  input/UI is Phase 5). No new env vars / feature flags.

## 2. The design

### 2.1 `src/sim/environment.ts` (NEW, PURE, TESTED) — the one real seam
```ts
export type TimeOfDay = "day" | "dusk" | "night";
export type Weather = "clear" | "rain" | "storm";

export interface Environment { time: TimeOfDay; weather: Weather; }

export interface EnvironmentParams {
  // physics
  mu: number;              // rail adhesion → resolveInputs (feeds real traction/brake caps)
  // visuals (consumed by scene.ts; all plain numbers/hex so they stay testable)
  skyColor: number;        // hex; also the fog colour
  fogNear: number;
  fogFar: number;
  ambientIntensity: number;// hemisphere light
  moonIntensity: number;   // directional ("moon"/"sun") light
  rainIntensity: number;   // 0..1 → particle opacity (& count scale); 0 ⇒ no rain
  railWetness: number;     // 0..1 → rail roughness (wet = lower roughness/sheen)
  wiperOn: boolean;        // wiper sweeps iff it's raining
}

/** The signature setting and the ring's first entry. */
export const DEFAULT_ENVIRONMENT: Environment = { time: "night", weather: "rain" };

export function environmentParams(env: Environment): EnvironmentParams;
export function cycleEnvironment(env: Environment): Environment; // steps PRESET_RING
```
- **μ model** `mu = clamp(weatherBase[weather] × timeFactor[time], ≥ 0.15)` with
  exact round numbers chosen so the default pins to 0.20:
  - `weatherBase`: `clear 0.30` (= `ADHESION.dry`), `rain 0.25`, `storm 0.20`.
  - `timeFactor`: `day 1.0`, `dusk 0.9`, `night 0.8` (cooler/greasier rail later).
  - **Calibration pin (exact):** night × rain = `0.25 × 0.8` = **0.20** =
    `ADHESION.wetNight`, so the default environment leaves physics **bit-identical**
    (no-regression). Spot values: clear-day 0.30, storm-night 0.16, clear-night
    0.24 — all monotone as ENV1 requires, all ≥ 0.15.
- **Visuals.** `night` darkest (low ambient/moon, deep blue sky), `dusk` mid
  (warmer, dimmer), `day` brightest (grey overcast when wet). `rainIntensity`:
  `clear 0`, `rain ~0.6`, `storm ~1.0`. `railWetness`: `clear 0.1`, `rain 0.8`,
  `storm 1.0`. `wiperOn = weather !== "clear"`. Fog tighter in storm.
- **`PRESET_RING`** (what the demo key cycles): `night+rain` (signature) →
  `night+clear` → `dusk+rain` → `day+clear` → back. `cycleEnvironment` returns the
  next ring entry; from any non-ring env it falls back to the ring head.

### 2.2 `src/render/scene.ts` (CHG, impure) — the world
- **RenderView gains `env: EnvironmentParams`** (projected in `main.ts`). Each
  frame `scene.render(view)` applies: `scene.background`/`fog` colour from
  `env.skyColor` + `fogNear/Far`; hemisphere `intensity = env.ambientIntensity`,
  moon `intensity = env.moonIntensity`; rain material `opacity = env.rainIntensity`
  (and **skip the rain scroll loop entirely when `rainIntensity === 0`** — perf);
  rail material `roughness = lerp(0.35, 0.08, env.railWetness)`; the cab wiper gated
  by `env.wiperOn` (via `CabView.wiperOn`).
- **Stations.** A `buildStation(stationCentreZ, side)` helper adds, per station:
  a platform slab (long thin box at platform height, offset in X to the running
  side), a canopy (flat roof on two/three posts), a back wall, an **emissive name
  board**, and 2-3 platform **lamps** (small emissive disc + a low-range
  `PointLight`, kept few — point lights are the main perf cost; budget ≤ ~2 per
  station). Built once. Use `route.stations[i].chainage` as Z, `platformHalf` for
  length.
- **Lineside scenery (instanced).** OLE masts every ~45 m along the route as one
  `InstancedMesh` (a vertical post + a short cantilever arm; the contact wire may
  be a single long thin line/box per track at pantograph height — cheap, optional).
  Fencing as an instanced low rail both sides. A handful of schematic **distant
  building** blocks set well back (instanced boxes, dark with a few emissive
  windows). **Mileposts** every ~500 m (tiny instanced posts). All sit at
  `z = chainage`, offset in X; none allocate per frame.
- **Coordinate conventions (so geometry lands in front of the camera, not behind
  it — the Phase-3 −Z trap):** world **+Z = toward Eastbank** (increasing
  chainage); the camera sits at `z = chainage − 0.6` looking toward `+Z`; lineside
  objects use `z = object chainage`, `x = ±offset` from track centre, `y ≥ 0` up.
  Track gauge centre is `x = 0`.

### 2.3 `src/render/cab.ts` (CHG, tiny) — wiper gate
Add `wiperOn: boolean` to `CabView`; when false, ease the wiper to its park angle
and stop advancing `wiperPhase` (clear weather ⇒ wipers off). No other change.

### 2.4 `src/main.ts` (CHG, impure) — wiring + the demo key
- Hold `let env = DEFAULT_ENVIRONMENT`. Each frame compute `const ep =
  environmentParams(env)`; pass `ep.mu` to `resolveInputs(controls, safety, ep.mu)`
  (was `ADHESION.wetNight`); include `env: ep` in the `RenderView`.
- A single **demo key** (`KeyE`) cycles the environment via
  `env = cycleEnvironment(env)` on its key-edge. This is *not* a `ControlIntent`
  (it drives no train control), so it's handled in `main.ts`'s key edge set
  directly, calling the **pure** `cycleEnvironment` — `main.ts` keeps no arithmetic.
  (It moves into the Phase-5 action model later; journaled.)

### 2.5 What stays pure / untested
- **Pure + tested:** `environment.ts` (`environmentParams`, `cycleEnvironment`).
- **Untested impure adapters (named):** the `scene.ts` station/scenery/weather
  additions, the `cab.ts` wiper gate, the `main.ts` wiring — Three.js, verified by
  build + headless screenshot, exactly as Phases 1 & 3.
- **`src/sim/physics.ts`, `simulation.ts`, `controls.ts`, `aws.ts`, `route.ts`
  data, `ui/hud.ts`, `audio/*` — UNCHANGED.** (`route.ts` may gain *no* fields;
  stations/signals already carry the chainages scenery needs.)

## 4. Acceptance criteria (definition of done)

- **ENV1** Wet conditions reduce adhesion: `environmentParams({any,rain}).mu` and
  `{any,storm}.mu` `<` `{same time, clear}.mu`; storm `<` rain.
- **ENV2** Night is darker than dusk is darker than day: `ambientIntensity` and
  `moonIntensity` strictly increase night → dusk → day (same weather).
- **ENV3** `rainIntensity > 0` and `wiperOn === true` iff `weather !== "clear"`;
  `clear` ⇒ `rainIntensity === 0` and `wiperOn === false`.
- **ENV4** **Calibration pin:** `environmentParams(DEFAULT_ENVIRONMENT).mu` ===
  `ADHESION.wetNight` (0.20) within 1e-9 — the default leaves physics unchanged.
- **ENV5** All `environmentParams` outputs finite (no NaN/Inf) and μ ≥ 0.15 across
  the full `TimeOfDay × Weather` domain; `railWetness`/`rainIntensity` ∈ [0,1].
- **ENV6** `cycleEnvironment` walks the preset ring and returns to
  `DEFAULT_ENVIRONMENT` after `PRESET_RING.length` steps; every result is a valid
  `Environment`.
- **REG** the existing **111** tests still pass unchanged; `tsc` zero warnings.
- **BUILD** `npm run build` clean.
- **SMOKE** (headless, zero console/page errors): a screenshot shows the wet-night
  world with **station platforms + canopy + name board** and **lineside scenery
  (OLE masts)**; a second screenshot **after cycling the environment** to a
  brighter/clear preset shows visibly **changed lighting and no rain**.
- **NO-REGRESSION** train still drives (power moves it; HUD aspect updates); with
  the default environment the drive is physically identical to Phase 3 (ENV4).

## 5. Residual risks
- **Aesthetic quality not autonomously verifiable** — "renders + plausible
  screenshot" ≠ "looks like a railway." Flagged for human review.
- **Perf:** point lights (station lamps) and added instanced geometry must stay
  light; budget station lamps tightly and keep one InstancedMesh per scenery class.
  Verify the build still runs at interactive frame rate (sanity, not an oracle).
- **Geometry-behind-camera trap** (Phase-3 −Z): mitigated by the explicit
  coordinate convention in §2.2 and the screenshot gate.
- **μ calibration drift:** ENV4 pins the default; if a future tune moves
  `ADHESION.wetNight`, ENV4 must move with it (intentional coupling, documented).

## 6. Decision log
- **D1** Weather is a **physics** input (μ), not just a look — the one seam worth a
  pure tested module; everything else is schematic Three.js behind the screenshot.
- **D2** Default environment pins μ = 0.20 so Phase 0-3 physics/tests are untouched
  (ENV4 differential), de-risking the whole phase to "additive."
- **D3** Author HLD directly + self-sign-off (proportionality; Phase-3 precedent).
- **D4** Schematic world (instanced primitives + emissive + sheen), **not**
  photoreal/RTT — fits scope + perf budget.
- **D5** One-key demo cycle for environment now; fold into the Phase-5 action model
  later. Keep `main.ts` arithmetic-free by cycling via the pure `cycleEnvironment`.
- **D6** Don't lengthen the line; enrich the existing segment (scope discipline).
