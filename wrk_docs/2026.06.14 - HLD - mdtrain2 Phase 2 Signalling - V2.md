# HLD -- mdtrain2 Phase 2: Signalling

Status: under review (V2) ¬∑ Date 2026-06-14 ¬∑ Target `C:\language\mdtrain2` (Phase 0+1 merged, verified at HEAD) ¬∑ `package.json` 0.2.0 ‚Üí 0.3.0

This design adds a believable signalling / AWS / TPWS / SPAD layer to WESTFORD‚ÜíEASTBANK by **growing state behind `tickAws`** and making **one scoped edit to `tickSafety`**: the carry-forward seed at `controls.ts:258` is narrowed to **DSD-only**, so AWS/TPWS reasons are re-derived fresh from the fold every tick and become releasable. The release machine (`tickSafety` `:277-281`), `resolveInputs`, and `reduceControls` are **byte-for-byte unchanged**. It is written testability-first: ¬ß2.0 derives the function shapes from the oracle list (¬ß4), so the tested surface is minimal and traceable; the untested surface is two DOM lamps plus one line of `prevChainage` plumbing and one line of `dir` plumbing.

**The signal model (revised V2, F2).** Each station carries a **STARTER signal positioned just past its stop board** (the departure end). The starter is held **RED until the station is served** ‚Äî the driver stops within the platform at a stand, which marks the station served and clears that starter. An **AWS magnet** sits ~180 m on approach to each starter. Signals further back show single-yellow / double-yellow by looking ahead to the next red(s). The normal cycle is therefore: *approach ‚Üí AWS warning for the red starter ahead ‚Üí ack ‚Üí stop at the platform & serve ‚Üí starter clears ‚Üí depart.* A SPAD / TPWS-TSS fires **only** if the driver passes the red starter **without having served** (a blow-through). A normal served stop is SPAD-free by construction, because serving and the starter-clear both happen at the platform, *before* the front reaches the starter post just beyond it.

## 1. Scope and non-goals

### In scope
- **Route data**: fixed STARTER signal positions on `Route`, authored on WESTFORD_EASTBANK as **point features** (one chainage per starter post = `station.chainage + STARTER_OFFSET`; AWS-magnet and OSS-loop chainages are *derived* off the post chainage, not stored separately).
- **4-aspect colour-light aspects** (`RED` / `YELLOW` / `DOUBLE_YELLOW` / `GREEN`) derived by a pure one-signal-ahead cascade, each starter held RED until its station is served. "Section occupied" is proxied by **a station with an unserved booked stop** (single player; no second train).
- **AWS**: a non-green magnet on approach ‚Üí caution + sunflower ‚Üí driver acknowledges within a `dt`-counted window, else a full-service penalty brake. CLEAR/WARNING/BRAKE machine grown behind `tickAws`, returning `PenaltyReason "AWS"`.
- **TPWS**: TSS (train-stop, passing a red starter unserved) and OSS (overspeed at a loop ahead of a red starter) ‚Üí `PenaltyReason "TPWS"`.
- **SPAD** detection (front passing a RED starter that has *not* been served) ‚Üí asserts `"TPWS"` (the TSS train-stop *is* the same physical crossing) plus a sticky `spad` flag for HUD/scoring; **no new union member**.
- **Latch enabling change (P2-RA)**: scope the `tickSafety` carry-forward seed (`controls.ts:258`) to **DSD-only**, so AWS/TPWS reasons live in the set **iff `tickAws` returns them this tick**. This is the one-line edit that makes the AWS/TPWS latch releasable. The release machine stays untouched.
- **HUD**: current signal aspect + an AWS sunflower indicator (new `HudView` fields, projected purely; two new DOM lamps).
- **Pure tested core**: all AWS/TPWS/aspect logic advances off explicit `dt`; nothing under `src/sim` uses `Date`/`Math.random`/`performance.now`/`setTimeout`. Penalty arrives **only** as a `PenaltyReason` through the latch (full service) ‚Äî never via `SimInputs.emergency`.

### Non-goals (deferred ‚Äî do not build)
Interlocking ¬∑ route-setting ¬∑ junctions/points ¬∑ second train ¬∑ ATP/ETCS ¬∑ timetable/dwell engine (no `dwellUntil`) ¬∑ real block-occupancy detection (we use the served-stop proxy) ¬∑ audio from `src/sim`. `SimInputs.dir` is **not** widened to neutral. No env vars, feature flags, or build steps.

**`served` is a one-way "booked stop done" flag (F7, deferred).** It is monotonic add-only: once a station is served it stays served for the run. Reversing back and re-approaching a served station does **not** re-arm its starter (the starter stays GREEN, no fresh SPAD risk). This is the realistic "the booked stop is done" semantics for a single-player one-shot run; a re-arm-on-reverse model is out of scope and unneeded for the route's one-directional booked stops. (One-line note ‚Äî no special handling, no oracle.)

## 2. The design

### 2.0 Shape, derived from the oracles (testability-first)

The oracle list (¬ß4) demands exactly **four pure functions** plus **one threaded state record**. The design is shaped to *that* and nothing more:

| Oracle group | Function under test | Inputs it forces |
|---|---|---|
| **A** aspect cascade | `aspectAt(route, i, served)` | route starters + served-set, **no train, no `dt`** |
| **B** AWS warn‚Üíack / warn‚Üípenalty | `tickAws` phase machine | magnet crossing, `intent.acknowledge`, `dir`, `dt` |
| **C** penalty survives ack until source clears, then releases | `tickAws` latch lifecycle + DSD-scoped seed + fold order + one-tick lag | reason re-asserted on the ack-at-stand BRAKE tick, dropped the *next* tick |
| **D** TPWS TSS / **F** SPAD | `tickAws` crossing test | red starter crossed unserved |
| **E** TPWS OSS | `tickAws` crossing test | OSS loop crossed at overspeed behind a RED starter |
| **G** dt-slice independence | all of the above | coarse vs fine `dt` end-state equality (served held constant) + reverse guard |
| **H** O13 migration | `tickAws` stub‚Üíreal | empty-signal route ‚áí no reasons |

Everything timed lives in **`AwsState`**, owned by `aws.ts`, threaded by `main.ts`. `SafetyState` absorbs **no** AWS internals ‚Äî only `PenaltyReason` crosses the boundary.

### 2.1 Route signal/AWS data model (`route.ts`)

Starters are **point features**, not piecewise-constant fields ‚Äî they get their own sorted array and point queries. **Do not** force them into `Segment<T>` (that abstraction is `[from, to)` intervals; misusing it is the M8-on-ramp trap). Aspects are **derived**, never stored.

```ts
// route.ts ‚Äî NEW
export type Aspect = "RED" | "YELLOW" | "DOUBLE_YELLOW" | "GREEN";

export interface Signal {
  /** Starter-post chainage, m. The single stored chainage.
   *  By authoring rule = its station's stop-board chainage + STARTER_OFFSET
   *  (the post sits JUST PAST the platform, at the departure end). */
  chainage: number;
  /** Station whose unserved booked stop holds this starter RED.
   *  ("" = plain block signal, never self-held; red only via the ahead cascade.) */
  protects: string;
}

export interface Route {
  length: number;
  stations: Station[];
  grades: Segment<number>[];
  speedLimits: Segment<number>[];
  curvatures: Segment<number>[];
  signals: Signal[]; // NEW ‚Äî required; sorted ASCENDING by chainage (authoring invariant)
}
```

- **One chainage per starter post.** The AWS magnet (`chainage ‚àí AWS_MAGNET_OFFSET`) and the OSS loop (`chainage ‚àí OSS_LOOP_OFFSET`) are **derived** at use-site, never stored. This is the leanest believable shape. Constants live in `aws.ts`; **`STAND_EPS` is not overloaded**.
- `signals` is **required** (house style: required field + explicit default, cf. `SimInputs.emergency`). The three synthetic fixtures get `signals: []`, which keeps every Phase-0/1 oracle green and makes the O13 migration trivial (empty signals ‚áí `tickAws` returns no reasons).
- **No `blockSections`.** The served-stop proxy needs none; adding interval block data now would be premature (D7).

Pure point queries, in `route.ts`, mirroring `lookup`'s guarded style:

```ts
/** Nearest starter AHEAD of s in travel direction dir, or null. */
export function nextSignalAhead(route: Route, s: number, dir: 1 | -1): { i: number; sig: Signal } | null;

/** Indices of any positions in `points` that lie in the forward swept interval
 *  (sPrev, sNow]. Generic over an arbitrary position array so one query serves
 *  starter posts, AWS magnets, and OSS loops alike (F4). */
export function pointsCrossedFwd(points: number[], sPrev: number, sNow: number): number[];
```

`pointsCrossedFwd` (F4) **replaces** the V1 signal-only `signalsCrossedFwd`. It is purely geometric over a number array, so the caller can query three derived arrays in one place ‚Äî starter posts, posts‚àí`AWS_MAGNET_OFFSET`, posts‚àí`OSS_LOOP_OFFSET` ‚Äî and the indices it returns map straight back to the starter index `i` (same ordering). This handles AWS-magnet crossings, OSS-loop crossings, and large-`dt` swept intervals uniformly. Direction reconciliation lives in `tickAws` (¬ß2.4); `pointsCrossedFwd` only handles the forward case (`sNow > sPrev`) ‚Äî reverse/standing crosses nothing. Both queries are end-clamp-safe and array-index-guarded for `noUncheckedIndexedAccess`.

**WESTFORD_EASTBANK authoring (revised, F2).** Each served station gets a starter just past its platform; Westford (the origin, never a booked *arrival*) needs no protecting starter. With `STARTER_OFFSET = 60`, `AWS_MAGNET_OFFSET = 180`, `OSS_LOOP_OFFSET = 50`:

| Starter | station | stop board | post chainage (+60) | magnet (‚àí180) | OSS loop (‚àí50) | rationale |
|---|---|---|---|---|---|---|
| S1 | Riverside | 1500 | 1560 | 1380 | 1510 | magnet falls in the 25 mph Riverside approach (1400‚Äì1600) |
| S2 | City Centre | 3100 | 3160 | 2980 | 3110 | **long run-in** (S1‚ÜíS2 ‚âà 1600 m) ‚Üí believable G‚ÜíYY‚ÜíY‚ÜíR |
| S3 | Victoria Street | 4400 | 4460 | 4280 | 4410 | magnet in the 25 mph Victoria approach (4300‚Äì4500) |
| S4 | Eastbank | 6000 | 6000 | 5820 | 5950 | terminus starter clamped to route end; 15 mph buffer run-in |

Notes: S4's post would be 6060 m but is clamped to `route.length` (6000) ‚Äî the buffer-stop terminus has no departure beyond it; this does not affect the SPAD rule (you serve Eastbank before reaching the buffers). Each post sits **past** its platform, so a train that stops within `¬±platformHalf` of the stop board serves the station (and clears the starter) while still short of the post ‚Äî the normal stop never crosses a RED starter. The S1‚ÜíS2 ~1600 m gap is the one section long enough that double-yellow is *not* decorative; closer pairs show only R/Y/G (acceptable and realistic).

### 2.2 Aspect rule (pure, train-free)

A starter `i` is **held RED** iff its `protects` station is **not in `served`** ‚Äî the single-player "section occupied / booked stop outstanding" proxy. Otherwise it cascades off **one** signal ahead. `served` lives in `AwsState` (¬ß2.3). The rule tests a signal's **own held-red FIRST**, then looks ahead:

```ts
// route.ts ‚Äî pure, O(1) per signal amortised, no train, no dt
export function aspectAt(route: Route, i: number, served: ReadonlySet<string>): Aspect {
  const sig = route.signals[i];
  if (!sig) return "GREEN";                                   // guard (noUncheckedIndexedAccess)
  if (sig.protects !== "" && !served.has(sig.protects)) return "RED"; // OWN held-red first
  const ahead = route.signals[i + 1];                         // next starter toward Eastbank
  if (!ahead) return "GREEN";                                 // last signal, nothing ahead
  const aheadAspect = aspectAt(route, i + 1, served);         // backward sweep, depth ‚â§ n
  if (aheadAspect === "RED") return "YELLOW";
  if (aheadAspect === "YELLOW") return "DOUBLE_YELLOW";
  return "GREEN";
}
```

Recursion depth is bounded by `signals.length` (4); index strictly increases ‚áí no cycles; terminates at the last signal (oracle A4 pins the guarded out-of-range read). Double-yellow is reachable and meaningful: with only S4 unserved, S4=RED, S3=YELLOW, S2=DOUBLE_YELLOW, S1=GREEN.

**Served-set writer.** `served` grows in the `tickAws` path: a station enters `served` the first tick the train front is within `¬±platformHalf` of its stop board **and** `|speed| ‚â§ STAND_EPS`. This is the only writer; it is a position-only proxy, not a dwell model (Residual #2), and is **monotonic add-only** (F7). It is also the one `dt`-sensitive write (G-note, ¬ß4 G).

### 2.3 `AwsState` + the new `tickAws` signature and state machine

Per the brief's mandated signature, `prevChainage` is passed **explicitly** (not folded into `AwsState`), and ‚Äî new in V2 ‚Äî `dir: 1 | -1` is passed **explicitly** (F5): at a stand `sign(state.speed)` is undefined, so heading must come from the same authority `resolveInputs` already uses, `ControlState.lastDir`. `main.ts` derives `dir` from `controls.lastDir` and passes it in; `tickAws` stays pure on its explicit arguments.

```ts
// aws.ts ‚Äî owned here, threaded by main.ts; pure, dt-counted
export type AwsPhase = "CLEAR" | "WARNING" | "BRAKE";
export type Sunflower = "BLACK" | "CAUTION"; // all-black = clear; black/yellow = caution-latched

export interface AwsState {
  phase: AwsPhase;        // AWS warning/brake machine
  warnTimer: number;      // s remaining in the ack window; counts DOWN off dt (WARNING only)
  sunflower: Sunflower;   // latched caution reminder; reset only by a GREEN magnet
  brakeReason: PenaltyReason | null; // single BRAKE-latch source: "AWS" | "TPWS" | null
  releaseArmed: boolean;  // F1: set on the ack-at-stand BRAKE tick; next tick transitions to CLEAR
  served: ReadonlySet<string>;       // stations whose booked stop is done (occupancy proxy)
  spad: boolean;          // sticky SPAD event flag (HUD/scoring)
}

export interface AwsHud { sunflower: Sunflower; spad: boolean; } // F8: NO aspect field here
export interface AwsOutput { reasons: PenaltyReason[]; }         // UNCHANGED latch-boundary type

export function createInitialAws(): AwsState {
  return { phase: "CLEAR", warnTimer: 0, sunflower: "BLACK", brakeReason: null,
           releaseArmed: false, served: new Set(), spad: false };
}

export function tickAws(
  aws: AwsState,
  state: SimState,
  route: Route,
  intent: ControlIntent,
  prevChainage: number,
  dir: 1 | -1,          // F5: explicit heading (from ControlState.lastDir in main.ts)
  dt: number,
): { next: AwsState; reasons: PenaltyReason[]; hud: AwsHud };
```

**One BRAKE latch path for AWS + TSS + OSS.** Rather than two booleans, a single `phase === "BRAKE"` plus a one-field `brakeReason: "AWS" | "TPWS" | null` carries the latched demand. Both an AWS-deadline miss and a TPWS TSS/OSS trip set `phase = "BRAKE"` and `brakeReason`; both re-assert *their own* reason every BRAKE tick and both clear identically through the one-tick release lag below (D5).

#### The state machine (one tick, all `dt`-counted)

Front position uses a fixed `FRONT_OFFSET` (legacy 2.0 m) **in the travel direction**: `frontNow = state.chainage + dir¬∑FRONT_OFFSET`, `frontPrev = prevChainage + dir¬∑FRONT_OFFSET`. **All edge logic fires only on strict forward motion `frontNow > frontPrev`** ‚Äî reverse or standing crosses nothing (D10), so a reversing train never false-triggers AWS/TPWS/SPAD.

1. **Served update**: if `|speed| ‚â§ STAND_EPS` and front within `¬±platformHalf` of a station, add it to `served` (copy-on-write, add-only). Clears that station's starter (and relaxes the cascade behind it).
2. **Magnet crossings** (`pointsCrossedFwd` over the posts‚àí`AWS_MAGNET_OFFSET` array): for each magnet crossed, evaluate that starter's `aspectAt` over the (possibly just-updated) `served`:
   - **GREEN magnet** ‚Üí `sunflower = "BLACK"`; if `phase === "WARNING"`, cancel to `CLEAR` (real-AWS clear bell). Does **not** clear a `BRAKE` (F6: guarded on `phase !== "BRAKE"`).
   - **non-GREEN magnet** ‚Üí only if `phase !== "BRAKE"` (F6): `phase = "WARNING"`, `warnTimer = AWS_WARN_WINDOW`, `sunflower = "CAUTION"`. A latched BRAKE outranks a fresh magnet warning ‚Äî a new non-green magnet can neither override nor re-arm an existing BRAKE latch.
3. **WARNING countdown**: if `phase === "WARNING"`:
   - `intent.acknowledge` (allowed **while moving** ‚Äî D4) ‚áí `phase = "CLEAR"`, sunflower stays `CAUTION` (latched reminder), no penalty.
   - else `warnTimer ‚àí= dt`; if `warnTimer ‚â§ 0` ‚áí `phase = "BRAKE"`, `brakeReason = "AWS"`.
4. **TPWS / SPAD** (`pointsCrossedFwd` over the starter-post array, evaluating `aspectAt` at the crossing):
   - **TSS / SPAD**: a starter crossed while `aspect === "RED"` (i.e. its station **unserved** ‚Äî a blow-through) ‚áí `phase = "BRAKE"`, `brakeReason = "TPWS"`, `spad = true` (the train-stop trip *is* the SPAD). A served station's starter is GREEN/clear, so a normal stop crosses nothing red ‚Äî SPAD-free.
   - **OSS**: the starter's OSS loop crossed while that starter `aspect === "RED"` **and** `|speed| > OSS_TRIP_SPEED` ‚áí `phase = "BRAKE"`, `brakeReason = "TPWS"`.
   - **Within-tick precedence (F12)**: when a single coarse `dt` sweeps past *both* a magnet (step 2) and its red starter (step 4), AWS is evaluated first (step 2/3 enter WARNING, decrement, may fall to BRAKE on `brakeReason="AWS"`), then TPWS/TSS (step 4) runs and **overwrites `brakeReason` to `"TPWS"`** for the same physical crossing and sets `spad`. Deterministic order: *served ‚Üí magnet/AWS ‚Üí TPWS/SPAD ‚Üí release*. TSS/SPAD is the authoritative cause when both fire in one tick (the train physically passed the red), so `"TPWS"` wins; the latch behaviour (full-service brake) is identical either way.
5. **BRAKE release ‚Äî ONE-TICK LAG (F1).** While `phase === "BRAKE"`:
   - If **not** `releaseArmed` and `intent.acknowledge && |speed| ‚â§ STAND_EPS`: this is the **ack-at-stand tick**. **Stay in BRAKE this tick**, set `releaseArmed = true`, and **RE-ASSERT** `brakeReason` (so step 6 returns `[brakeReason]` and `tickSafety`'s fold re-adds it *this* tick ‚Äî C1/C3 survive). This records "release requested" without yet clearing.
   - Else if `releaseArmed` (the **next** tick, regardless of further input): transition `phase ‚Üí CLEAR`, `brakeReason ‚Üí null`, `releaseArmed ‚Üí false`, sunflower stays `CAUTION` until the next GREEN magnet. Step 6 now returns `[]`, so the DSD-scoped seed drops the reason ‚áí the latch releases (C2/C2-PIN).
   - Else (in BRAKE, no qualifying ack-at-stand, or ack while moving ‚Äî C4): re-assert `brakeReason`, stay in BRAKE. Ack-while-moving never arms the release, so it never clears.
   - (`spad` stays sticky ‚Äî a scored event.)
6. **Reasons assembled**: `reasons = brakeReason ? [brakeReason] : []`. (Because step 5 keeps `brakeReason` non-null on the ack-at-stand tick and only nulls it the next tick, the survive/clear sequence is observable: assert ‚Üí ‚Ä¶ ‚Üí ack-at-stand tick still `[brakeReason]` ‚Üí next tick `[]`.)
7. **HUD**: return `{ sunflower, spad }` **only** (F8). The HUD aspect is **not** computed here; `buildHudView` derives it from `served` via `nextSignalAhead` + `aspectAt` (¬ß2.6).

All countdowns are `dt`-subtractive and all crossings use swept intervals, so coarse and fine `dt` reach equal end-states for everything **except** the served-set sample (the one `dt`-sensitive write ‚Äî G-note). A coarse `dt` that jumps a magnet *and* its starter in one tick is handled because steps 2 and 4 consult the same swept interval, in the F12 order.

**Ack split (D4 / F9).** The WARNING-window ack (cancel-before-penalty) is internal to `tickAws` and allowed while moving; the latch-release ack (arming the one-tick release of an *asserted* reason) is ack-at-stand. **Both ‚Äî plus the DSD/penalty release in `tickSafety` ‚Äî consume the single `intent.acknowledge` edge (F9).** This dual/triple action on one ack is **intended and realistic**: a real driver presses one acknowledge that both cancels the AWS warning and releases a stand-still penalty. No separate ack path, no separate handling.

### 2.4 How reasons re-assert to survive ack, and when they stop (load-bearing)

This is the keystone mechanism. It depends on **two** facts in `tickSafety`: the carry-forward seed at `controls.ts:258` is **scoped to DSD-only**, and the fold at `:275` runs **before** the DSD-only delete at `:278-281`. The DSD-scoping is the **enabling change (P2-RA)** without which the AWS/TPWS latch is permanently un-releasable.

**The seed scoping (the one-line edit at `:258`):**

```ts
const reasons = new Set<PenaltyReason>();
if (s.penaltyReasons.has("DSD")) reasons.add("DSD");
```

This replaces the original `const reasons = new Set<PenaltyReason>(s.penaltyReasons);`. After this edit, AWS/TPWS reasons are **never seed-carried** ‚Äî they are re-derived **fresh from the fold** (`for (const r of aws.reasons) reasons.add(r)`, `:275`) on every tick. They are present in the set **iff `tickAws` returns them this tick**.

- **Survives ack (C1/C3).** While `phase === "BRAKE"`, `tickAws` re-asserts `brakeReason` **every tick ‚Äî including the ack-at-stand tick** (the one-tick lag, F1): on that tick `tickAws` arms the release but still returns `[brakeReason]`, so the fold (`:275`) re-adds it. The DSD-only delete at `:278-281` cannot touch it. So an ack-at-stand that empties DSD does **not** strand-clear the AWS/TPWS reason on the ack tick ‚Äî the fold re-adds it the same tick. This is exactly the pre-authorised "source still demands it" clause (`controls.ts:244-246`).
- **Stops / clears cleanly (C2).** The reason drops the first tick `tickAws` **stops** returning it ‚Äî which, by the one-tick lag, is the tick **after** the ack-at-stand tick. On that next tick `tickAws` transitions `phase ‚Üí CLEAR`, `brakeReason ‚Üí null`, and returns `[]`. Now ‚Äî **because the seed is DSD-scoped** ‚Äî the reason is no longer seed-carried, the fold adds nothing, and the set empties ‚Üí the latch releases. **This requires the `:258` seed-scoping**: under the original full carry-forward seed, the reason would be re-seeded every subsequent tick from `s.penaltyReasons` and never deleted (the release names only "DSD"), making the latch un-releasable forever.
- **Why the release machine stays untouched.** The original `:280` comment ("Phase-2 persisting reasons would re-add themselves above") assumed fold-sourced reasons are **not** seed-carried ‚Äî i.e. that a reason present in the set comes *only* from this tick's fold, not from yesterday's seed. The seed-scoping is precisely what makes that assumption true. With the seed DSD-scoped, the release machine's "delete DSD only" rule is already correct: every other reason in the set is, by construction, this-tick fold output, and drops automatically when the fold stops producing it. So the release machine (`:277-281`), `resolveInputs`, and `reduceControls` need **no edit** ‚Äî only the seed is scoped.
- **`tickAws` only ever touches its own reasons** ‚Äî never `"DSD"` (R1), and DSD logic never touches `"AWS"`/`"TPWS"`.

> **`tickSafety` release machine + signature unchanged; the carry-forward seed scoped to DSD-only (one line, `:258`).** Survive = the fold re-adds the reason after the DSD-only seed + DSD delete, every tick `tickAws` demands it (including the ack-at-stand tick, by the one-tick lag); clear = `tickAws` stops demanding it on the next tick **and** the DSD-scoped seed no longer carries it ‚áí it drops.

### 2.5 `tickSafety` + test-helper ripple

- **`tickSafety` release machine + signature are UNCHANGED; the carry-forward seed is scoped to DSD-only (one line, `:258`).** It still takes `aws: AwsOutput` and folds `aws.reasons` at `:275`. The release machine (`:277-281`), `resolveInputs`, and `reduceControls` are **byte-for-byte untouched** ‚Äî they inherit notch-0 + full-service + power-to-OFF for AWS/TPWS for free via `penaltyActive`. The **only** change inside `tickSafety` is narrowing the seed at `:258` so it carries forward DSD and nothing else.
- **`main.ts`** threads the new args and persists `AwsState` + `prevChainage`:
  ```ts
  let aws = createInitialAws();
  let prevChainage = state.chainage;
  // per frame, AFTER step():
  const dir = controls.lastDir;                 // F5: same authority resolveInputs uses
  const awsOut = tickAws(aws, state, route, intent, prevChainage, dir, dt);
  aws = awsOut.next;
  safety = tickSafety(safety, intent, state, dt, { reasons: awsOut.reasons });
  hud.update(buildHudView(state, controls, safety, route, aws.served, awsOut.hud));
  prevChainage = state.chainage;
  ```
  `main.ts` stays arithmetic-free (still the untested shell). The richer `tickAws` return is destructured to `{ reasons }` at the latch boundary, so only `PenaltyReason` crosses into the latch module (D2).
- **`test/controls.test.ts` ripple** (exact):
  - `tickFor` (`:77-89`) threads an `AwsState`: seed `createInitialAws()` before the loop; each step `const o = tickAws(a, state, route, it_, state.chainage, dir, dt); a = o.next; cur = tickSafety(cur, it_, state, dt, { reasons: o.reasons });`. `tickFor` gains `route` (default `flatRoute`, now `signals: []`) and `dir` (default `1`) params, so every DSD oracle stays green with `reasons: []`.

## 4. Acceptance criteria / test oracles (definition of done)

All `environment:node`, no DOM, no wall-clock. Synthetic routes built inline; helpers thread `AwsState` in the existing idiom. These are the contract ‚Äî done when they pass and nothing else regresses.

**A ‚Äî aspect cascade** (`aspectAt`, train-free; `aspectRoute` = the 4 starters of ¬ß2.1):
- **A1**: from a fixture where **only S4's station is unserved** (the other three served), assert S4=`RED`, S3=`YELLOW`, S2=`DOUBLE_YELLOW`, S1=`GREEN`. - **A2**: every station served ‚áí all starters `GREEN`.
- **A3**: only S2's station unserved (S1, S3, S4 served) ‚áí S2=`RED`, S1=`YELLOW`, S3=S4=`GREEN` (single-step caution, isolated red).

**B ‚Äî AWS warning machine** (`awsFor`; magnet at S1's `1560‚àí180=1380`, S1 station unserved ‚áí S1 RED):

## 5. Residual risks

1. **Fold-before-delete order + seed-scoping + one-tick-lag coupling (highest).** The persistence-and-release mechanism rests on three coupled facts: the seed at `:258` is **DSD-scoped**; the fold (`:275`) runs **before** the DSD-only delete (`:279`); and `tickAws` applies a **one-tick release lag** (re-asserts on the ack tick, nulls the next). The un-releasable-latch risk is resolved by the seed-scoping; the ack-strand risk is resolved by the one-tick lag re-assertion. What remains fragile is **fold-order**: moving the fold after the delete, or generalising the delete to non-DSD reasons, would mis-clear mid-demand. C1/C2/C2-PIN/C3 are the guards; their comments must cite `:258`, `:275`, `:279` and the F1 lag.
2. **Served-set as occupancy proxy is a fiction, and is `dt`-sensitive.** Aspect colour couples to "drove well enough to stop." The exact tick a station flips to served depends on `dt` slicing (G-NOTE) ‚Äî pinned, defined, and excluded from the dt-independence claim. A train crawling through a platform without truly stopping could mis-flag. If it feels gamey, swap `served` for a scripted held-signal list with zero cascade changes.
3. **Double-yellow only on the S1‚ÜíS2 section.** On a 6 km route a driver rarely sees two reds ahead. A1 proves the logic; the route's *use* of YY is decorative-but-correct.
4. **OSS abstraction.** Speed-threshold-at-a-loop-chainage, not real loop-spacing/arm-timer physics. Honest abstraction, flagged in code; trip speed is one tunable constant.
5. **Direction reconciliation.** Heading now comes from explicit `dir = ControlState.lastDir` (F5), not `sign(speed)`, so a stand no longer has undefined heading. All crossings still gate on strict `frontNow > frontPrev`; D2/G2 guard the reverse and coarse-dt cases.
6. **`prevChainage` + `dir` plumbing is in the impure shell.** A wiring bug (post-step chainage as prev, or wrong `dir`) is untestable at `main.ts`. Mitigation: `tickAws` is pure on its explicit `prevChainage`/`dir`, so the *logic* is fully tested; only the two-line plumbing is trust-based (same risk class as the existing `dt` plumbing).
7. **Starter-just-past-platform geometry (new, F2).** The SPAD-free normal stop relies on the post sitting beyond `¬±platformHalf` of the stop board. `STARTER_OFFSET=60` vs `platformHalf` (90‚Äì100 m) means the post is **inside** some platforms' nominal half-length envelope ‚Äî but the train serves (and clears the starter) on first stop within `¬±platformHalf`, which happens before the *front* reaches the post for a normal arrival from behind. Authoring check: ensure `STARTER_OFFSET` keeps the post ahead of where a normally-stopped front sits; D1-SERVED pins the behaviour, but the offset is an authoring tunable to verify against the served/stop geometry.
8. **F7 reverse-and-re-approach (out of scope).** `served` is monotonic add-only; reversing onto a served station does not re-arm its starter. Acceptable for one-shot booked stops; documented, not built.
