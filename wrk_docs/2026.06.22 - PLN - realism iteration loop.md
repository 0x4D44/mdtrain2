# PLN — realism iteration loop

**Goal (Arthur):** evolve The Night Cab into a *realistic train simulator* — it should
**feel like driving a real train in the real world**. That spans **day / dusk / night**
and **clear / rain / storm**, not just the signature wet-night. The rubric below is
mine to evolve as we learn what "realistic" means in practice.

This is an **open-ended, taste-driven** target with **no machine oracle** (unlike the
320-test sim core). So the loop is built to *converge*, not wander: a rubric gives
direction and a termination signal, and **Arthur is the periodic judge** (he is the
"someone who matters").

## The loop

```
while not "feels real enough":
  capture()      # npx playwright test  → e2e/screenshots/*.png   (slow: ~25 s/shot)
  findings = workflow("realism-critique")   # fan-out art-director critics → ranked change list
  implement(findings.top)                    # I edit src/render/* (+ env lighting constants)
  re-render(affected) ; re-judge             # confirm the fix; commit green
  # milestone? → build from trunk + deploy /night-cab/ ; checkpoint Arthur for taste
```

- **Capture** is the `@playwright/test` harness (`e2e/night.spec.ts`): drives the real
  game headlessly at set-piece chainages, any time-of-day via `env` (E-presses:
  0 day/rain · 1 dusk · 2 night · 3 day/clear), optional drive / look-around, asserts
  zero console errors, writes full-page stills. Two sets:
  - **Iteration set** (~3–4 shots) for fast inner-loop checks (~1–2 min).
  - **Full set** (~17 shots, day+dusk+night) for milestone reviews (~13 min).
- **Critique** is the `realism-critique` Workflow: N critic agents, each reading the
  same stills through one rubric lens, returning structured findings; one synthesiser
  dedupes + ranks into a prioritised change list mapped to suspected files. Diverse
  eyes beat my single viewpoint.
- **Implement** is me (hands on the code). **Re-render + re-judge** confirms before
  committing — never claim a fix without seeing it.
- **Deploy is a milestone, not a loop step** — iterate locally; deploy once per
  milestone (build from the integrated trunk, swap the almanac bundle).

## Constraints (these shape the loop)

- **No oracle.** Realism is judged, not tested. Mitigate with the rubric + critic
  fan-out + Arthur's periodic taste checkpoints. I flag diminishing returns rather
  than polish forever.
- **Slow renders** (~25 s/shot, software WebGL + bloom). Keep the inner loop on a
  small iteration set; full sweeps only at milestones.
- **Procedural-only** (HLD constraints carry over): no new npm runtime deps, no
  committed image assets, no `Math.random` in the frame loop, perf tiered (PointLights
  hard-capped). The pure sim core (`src/sim/*`) stays untouched — **except** the
  tunable lighting constants in `environment.ts` (exposure, light colours/intensities,
  fog, sky, sunDir, nightFactor), which are fair game and unit-test-guarded.

## Rubric (v1 — evolvable)

A finding is scored **blocker / major / minor / polish**. We clear blockers+majors,
then judge.

1. **Lighting & atmosphere** — sky gradient & fog believable for each time-of-day;
   exposure right (day bright, night moody); bloom not overblown; light *direction*
   coherent (shadows/sheen agree with the sun/moon); dawn/dusk colour temperature.
2. **Scale & proportion** — buildings, signals, bridges, trees, platforms sized right
   vs the train and each other; eye height / FOV / speed *feel* like a real cab.
3. **Materials & surfaces** — steel, masonry, ballast, rail, foliage, road read as
   *themselves* under each light; not flat, plastic, or uniformly tinted.
4. **Geometry & world integrity** — no floating, clipping, z-fighting, gaps; props sit
   on the terrain; gauge clearance; placement & density plausible (not sparse/cluttered).
5. **Believability (the real-driver lens)** — does it *feel* like a real railway? What
   most breaks the illusion? What's a real lineside got that ours lacks (sensible
   element mix, distance haze, ground texture, horizon)?

**Known starting bias:** all prior work was night-focused — **daytime is likely the
weakest** and is the common real-driving case, so it gets priority early.

## Iteration log

- **I0 (2026-06-22):** loop machinery set up — generalised the harness to day/dusk/night
  (`env` field), authored the `realism-critique` workflow, captured the first day+dusk+
  night set. **First critique done** (6 agents, ~500k tok): night & dusk strong, **DAY is
  the weak spot**. Root cause the synthesiser found (critics only circled it): the diffuse
  fill `DirectionalLight` is pinned overhead and never re-aimed to `env.sunDir` (only the
  tier-gated shadow sun uses it) → no felt daytime sun → flat shadeless light. 17 ranked
  changes; full list in the workflow run output (`tasks/wx9c6lt8k.output`). Blockers:
  (1) light tracks sunDir + grazing day sun; (2) stop windows glowing at noon + daylight
  masonry. Majors: day fog haze, inverted-zenith fix, night bloom/floor, de-toy buildings,
  material contrast, OLE-dropper diagonal.
  - **Milestone 1 (daytime atmosphere) ✅ done + verified:** ranks 1+4+3 — `moon` light
    tracks `sunDir`; per-time zenith (bright day dome, dark night) from `nightFactor`;
    grazing day sun (`SUN_DIR_RAW.day.y` 0.92→0.6); tighter day fog for aerial perspective;
    paler day sky base; day ambient 1.8→1.5. All in `environment.ts` (lighting knobs) +
    `scene.ts`. **Result:** daytime ground now shows light/shade, distance hazes into the
    horizon, scene has depth — the "flat bright nothing" is gone. 322 oracles green; night
    unregressed.
  - **Milestone 2 (daytime buildings) ✅ done + verified:** rank 2. `buildScenery` returns
    the facade material; scene.ts fades `emissiveIntensity = 1.9 * nightFactor` so windows
    glow at dusk/night but go dark by day (no noon burn-through). Daylight masonry tints
    (brick/buff/stone/sandstone/render/brown) replace the charcoal ones. `buildFacade`
    reworked: smaller recessed panes (glassy blue-grey, reflect sky by day), reveal/sill
    shadow, floor coursing, lighter masonry base. **Result:** day buildings read as masonry,
    not charcoal monoliths; windows no longer glow at noon. Night skyline preserved (smaller
    windows × 1.9 ≈ old 0.62-window × 1.2 emission). 322 green. **Daytime now coherent
    (atmosphere + buildings) → first integrate+deploy point.** Still toy-ish for later:
    cuboid rooflines (rank 7), flat ground/ballast/rail (rank 8), OLE-dropper bar (rank 9).
