# HLD — mdtrain2 "Grand World": living terrain, a grand route, a push to realism — V1

**Status:** DRAFT (V1, ultracode synthesis) · Date 2026-06-15 · Target `C:\language\mdtrain2` (Phases 0-5 merged, v0.10.0). `package.json` 0.10.0 → **0.11.0**.
**Provenance:** synthesised by the team lead from a 3-angle draft (simplest-viable / risk-first / testability-first) + a 3-judge panel. **Winner: testability-first (2 of 3 judges)** — best fit to the binding brief (testability + holdable simplicity), spine adopted whole. Grafts: risk-first's numeric budget/zero-alloc oracles, eased cab attitude, and eye-clearance clamp; simplest-viable's blend-field framing and gradient sky-dome baseline. Lead tie-breaks: reject C1 transition curves (keep C0/G1 + eased felt-smoothness) and reject chunked streaming terrain (build-once ribbon + far-horizon band) — both per the brief's binding simplicity.

The world is a flat green plane the rails lie *on*. This overhaul makes it a country the line **threads through** — cuttings that cut, embankments that carry, a river valley spanned by a **viaduct**, a headland bored by a **tunnel** — on a longer, seven-chapter **Grand Route**, lit with ACES tone-mapping, PBR materials, one real sun and a real sky. The governing instinct: **the world is geometry, and geometry is math.** Every metre of track, every terrain height, every prop's resting altitude, every "is this span a viaduct?" decision is a *pure function of route data*, computed and checked on the Node side with no GPU. Three.js becomes a thin, named, screenshot-verified projector that draws what the pure layer computed.

## 1. Scope & non-goals

### In scope
1. **A pure parametric centreline** (`src/sim/centerline.ts`, NEW, PURE, TESTED): `centerlineAt(route, s) → Frame{x,y,z, tangent, up, cant}`. Height is the **analytic integral of grade**; lateral offset is the **analytic integral of curvature** (circular-arc geometry). C0-continuous position, G1-continuous (kink-free) tangent. **The prime seam** — every placement derives from it.
2. **A pure terrain model** (`src/sim/terrain.ts`, NEW, PURE, TESTED): `terrainHeight(route, z, x)` blends the rail **formation height** (centreline y + ballast shoulder near the track) with a deterministic seeded-value-noise **natural ground** by lateral distance `|x|`. The signed difference `natural − formation` *is* the scenery: cuttings, embankments, river valleys, hills. Plus the pure predicates `viaductSpanAt`/`tunnelBoreAt`, the prop-anchoring invariant `anchorY`, and the eye-clearance clamp `clampEye`.
3. **A pure camera/cab frame** (`src/sim/camera.ts`, NEW, PURE, TESTED): `cameraFrame(route, s, look, eye) → {position, yaw, pitch, roll}` and `cabAttitudeTarget(cant, grade, scale) → {pitch, roll}` — all the camera trig that lived inline in `scene.ts` moves into a tested seam. The cab rides pitch + a **clamped, scaled** roll *target*; the render layer eases the live attitude toward it (§2.5).
4. **A real living landscape** rendered from the above: a **track-following terrain ribbon** (heightfield), a **multi-arch viaduct**, a **tunnel** bore + portals, **sea**/**river** water planes, all props **anchored** to the terrain. The flat `PlaneGeometry` ground, the cone "hills" and the faked `addBank` cuttings in `scenery.ts` are **deleted**.
5. **A push toward realism, free-first, tier-gated:** `ACESFilmicToneMapping` + exposure (free), an sRGB-correct pipeline, **procedural canvas textures + normal maps** (grass/ballast/rock/water — *no committed image assets*), a **gradient sky dome** reading the env palette + a **PMREM env map** for PBR reflections, and — **gated by the quality tier** — **one** shadow-casting sun and an **optional** bloom pass via `three/examples`. Everything scales down through `quality.ts`; nothing breaks if it's all off.
6. **A grand multi-chapter route** (`src/sim/route.ts`, CHG, data): `KINGSGATE_SEAHAVEN`, ~**14 km**, **7 chapters** (city → suburb → countryside → viaduct → coast → tunnel → sea terminus), with optional per-route `viaducts[]`/`tunnels[]`/`terrainSeed` bands the pure terrain layer reads.

### Non-goals (ruthless cuts)
- **No 2-D track plan / free bezier layout.** Curvature *integrates* to a lateral offset (closed form); that is all the eye needs. (D2)
- **No physics change.** `physics.ts`/`simulation.ts` keep using `gradeAt` on the 1-D scalar chainage; the 3-D path is render-only and **physics never imports it** (O17). The 164 tests and the sim core stay untouched.
- **No chunked/streaming/quadtree terrain.** A 14 km hand-authored line needs one build-once ribbon + a cheap static far-horizon band + fog — not a streaming subsystem. (D8; rejected risk-first's chunking.)
- **No C1 transition (easement) curves.** The centreline is C0-position / G1-tangent; the **eased cab attitude** supplies *felt* smoothness across breakpoints without geometry machinery. (D11; rejected risk-first's VT_RAMP.)
- **No water simulation, no real tunnel interior modelling** (fog + portal light is enough), no god-rays, no day/night cloud sim.
- **No new npm dependency** (`three/examples/jsm/*` sub-imports are part of `three@0.183`). **No env vars, no feature flags** — device capability → the pure quality tier is the only branch.
- No migration / back-compat (dev sandbox): the old `WESTFORD_EASTBANK` is replaced as the live route (kept only if a test still references it).

## 2. The design

### 2.0 Architecture in one breath
```
route data (1-D bands: grades, curvatures, speedLimits, signals, + optional viaducts/tunnels/terrainSeed)
   │
   ├─ physics.ts ──────────── gradeAt(route,s)                         [UNCHANGED 1-D path]
   │
   └─ centerline.ts ── centerlineAt(route,s) → Frame{x,y,z,tangent,up,cant}   ← PRIME SEAM
        ├─ terrain.ts ── terrainHeight(z,x); reliefAt; viaductSpanAt; tunnelBoreAt; anchorY; clampEye
        └─ camera.ts  ── cameraFrame(route,s,look,eye); cabAttitudeTarget(cant,grade,scale)

environment.ts ── environmentParams(env) → {…, exposure, sunDir, sunColorPbr, bloomStrength}
quality.ts     ── qualityFor(env) → {…, shadowsEnabled, bloomEnabled, ribbonHalfWidth, terrainSegLen,
                                       terrainSubdiv, attitudeScale}; lodForDistance(d,tier); ribbonVertexCount(opts)

render/* (IMPURE, screenshot-only): read the pure layer, ease the cab attitude, place & draw meshes. No spatial math.
```
Everything left of `render/*` is pure, lives under `src/sim` (or `src/render/quality.ts`, already pure), contains **no banned token** (`Date|Math.random|performance.now|setTimeout|setInterval` — only `Math.random` of those is even tempting, and the noise uses integer hashing instead), and is covered by vitest oracles. The existing **G3 grep guard** therefore polices the whole spatial model for free.

### 2.1 `src/sim/centerline.ts` (NEW, PURE, TESTED) — the prime seam
```ts
export interface Frame {
  x: number; y: number; z: number;                 // world position of the rail-head centre at chainage s
  tangent: { x: number; y: number; z: number };    // unit forward direction (z-dominant)
  up: { x: number; y: number; z: number };          // unit up after cant roll
  cant: number;                                     // super-elevation roll, rad (banks into curves)
}
export function centerlineAt(route: Route, s: number): Frame;
export function heightAt(route: Route, s: number): number;          // formation height = ∫₀ˢ grade
export function lateralOffsetAt(route: Route, s: number): number;   // planar offset = ∫∫ curvature
export function headingAt(route: Route, s: number): number;          // tangent yaw about up, rad
```
- **Height** (exactly checkable): grades are piecewise-constant `Segment<number>` already in `route.grades`. `heightAt(s) = Σ grade_i·(seg overlap with [0,s])`, clamped to `[0,length]`. Piecewise-constant integrand ⇒ piecewise-**linear** integral ⇒ automatically C0 at every breakpoint. Analytic oracle (O1), differential vs `gradeAt` (O2), continuity (O3).
- **Lateral offset** (the curvy one): integrate piecewise-constant curvature `κ` twice — heading `ψ(s)=∫κ`, planar `x(s)=∫sinψ`, `z(s)=∫cosψ`. Each segment is a straight line (κ=0) or a **circular arc** (κ≠0), both closed-form: after `θ=κL`, lateral `x=R(1−cosθ)`, advance `z=R sinθ` with `R=1/κ`. Accumulate the running `(x,z,ψ)` pose at each breakpoint → continuity is automatic. ψ carries across breakpoints ⇒ tangent is continuous (G1) even though κ steps. Arc oracle (O5), tangent continuity (O4).
- **cant**: `cant(s)=clamp(CANT_GAIN·|κ(s)|·v_ref(s)², CANT_MAX)`, signed by turn direction, where `v_ref=speedLimitAt(s)` — a property of the *route*, deterministic, not of live speed (O6).
- **`up`/`tangent`** are the closed-form derivatives of position (no numeric differentiation). `Math.sin/cos/sqrt` are allowed (only `Math.random` is banned); no clock. Lives in `src/sim` so G3 guards it.
- **z-vs-chainage note:** on curves the honest planar `z(s)` is slightly less than `s` (chord < arc). We adopt the honest `z(s)` so curves genuinely bend the world; `physics.ts` integrates *speed* into the scalar chainage and never reads `centerlineAt`, so it is unaffected. The coupling is one-directional (render depends on sim; never the reverse).

### 2.2 `src/sim/terrain.ts` (NEW, PURE, TESTED) — the living land
```ts
export function formationHeight(route: Route, z: number): number;          // centreline y + ballast shoulder near track
export function naturalGround(route: Route, z: number, x: number): number; // seeded value-noise land + authored valleys/hills
export function terrainHeight(route: Route, z: number, x: number): number; // blend(formation, natural) by |x|
export function reliefAt(route: Route, z: number): number;                 // natural(z,0) − formation(z): >0 cutting, <0 embankment
export function viaductSpanAt(route: Route, s: number): boolean;           // reliefAt < −VIADUCT_THRESH
export function tunnelBoreAt(route: Route, s: number): boolean;            // reliefAt >  TUNNEL_THRESH
export function anchorY(route: Route, z: number, x: number, clearance: number): number; // terrainHeight + clearance
export function clampEye(route: Route, s: number, eyeX: number, eyeY: number): number;  // ≥ terrainHeight + MIN_CLEARANCE
```
**The blend (the heart):**
```
formation(z)        = heightAt(route, z)                              // rail top
natural(z,x)        = baseProfile(route, z, x) + valueNoise(seed, z, x)
w(x)                = smoothstep(0, BLEND_W, |x| − SHOULDER)          // 0 on the track → 1 far out (BLEND_W ≈ 60–120 m)
terrainHeight(z,x)  = lerp( formation(z) + ballastShoulder(|x|), natural(z,x), w(x) )
```
- `|x| ≤ SHOULDER` (≈ 4 m): terrain **is** the formation + ballast shoulder (`w=0` exactly) — the line always sits on a clean bed (O8).
- `|x| ≥ SHOULDER + BLEND_W`: terrain **is** the natural land (`w=1`).
- Between, a `smoothstep` handoff (no step at the track edge). Where `natural > formation` the blend climbs above rail level → **cutting walls**; where `natural < formation` it falls away → **embankment banks**. One function, both reliefs, governed by the sign of `natural − formation`. The Phase-4 `addBank`/cone-hill hacks are deleted.

**`naturalGround` (deterministic, banned-token-free):** seeded integer-lattice value noise (mulberry-style hash on `route.terrainSeed + (⌊z/CELL⌋, ⌊x/CELL⌋)`, bilinear, 2–3 octaves) for rolling texture, **plus** authored large features keyed off route data: a **river valley** = a smooth negative Gaussian dip centred on each `route.viaducts[].center`; a **hill** = a positive Gaussian bump centred on each `route.tunnels[].center`. Big set-pieces are *authored* (data-driven, testable); the noise only adds texture between them. No `Math.random`, no clock — pure (O9 determinism).

**Predicates** are pure thresholds on the same `reliefAt`: the viaduct exists *because* the valley is deep, the tunnel *because* the hill is high — single source of truth, disjoint by O9. **`anchorY`** turns "props float/sink" into a tested invariant (O7): every scenery class computes its base Y from `anchorY` at build time. **`clampEye`** stops a pitching camera ducking under the land on grades (O20).

### 2.3 `src/sim/camera.ts` (NEW, PURE, TESTED) — the eye/cab frame, roll fork resolved
```ts
export interface LookOffset { yaw: number; pitch: number; }      // existing LMB drag accumulators
export interface CamFrame { position:{x:number;y:number;z:number}; yaw:number; pitch:number; roll:number; }
export function cameraFrame(route: Route, s: number, look: LookOffset, eye: EyeOffset): CamFrame;
/** PURE target attitude the render layer eases TOWARD; scale=0 ⇒ {0,0} (mobile/reduced-motion). */
export function cabAttitudeTarget(cant: number, grade: number, scale: number): { pitch: number; roll: number };
```
- `cameraFrame` composes: `position = centerlineAt(s).{x,y,z} + eye offset rotated into the frame, then clampEye`; `yaw = headingAt(s) + look.yaw`; `pitch = atan(grade) + look.pitch`; `roll` from `cabAttitudeTarget`. **All the trig that lived inline in `render()` moves here, tested.**
- **The cab fork — DECIDED (D7): the cab rides pitch + a *clamped, scaled* roll, eased.** Pitch from grade is gentle (≤ ~1.5 % ⇒ ≤ ~0.9°) → on. Roll from cant is the gorgeous viaduct-bank but the sickness risk, so the **pure** `cabAttitudeTarget` returns `roll = clamp(scale·cant, ±ROLL_MAX)` (ROLL_MAX ≈ 4°) and `pitch = clamp(scale·atan(grade)+atan(grade)·…)` — bounded and tested (O16). `scale` comes from the quality tier (`attitudeScale`: ~0.35 high, **0 on coarse-pointer & reduced-motion**), so the comfort floor is a *proved* property: `cabAttitudeTarget(c, g, 0) === {0,0}`. The **ease** (advancing the live attitude toward the target by `approach()` each frame) is render-local state in `scene.ts` — stateful, so it stays out of `src/sim`. This gives a *felt* bank with machine-bounded amplitude and a provably flat mobile/reduced-motion path. The driver's LMB look offset adds on top, unchanged.

  shadowsEnabled: boolean; bloomEnabled: boolean;

- **reducedMotion:** keeps `rainCount=0` (unchanged) and forces `attitudeScale=0`.
- The existing `{pixelRatioCap, rainCount}` values are **byte-for-byte preserved** (no regression). New fields are additive. `ribbonVertexCount` makes "mobile must not break" a **numeric** oracle (O18), not just an ordering.

### 2.7 Render adapters (IMPURE, screenshot-verified) — no spatial math
- **`src/render/terrain-mesh.ts` (NEW).** Builds the **track-following terrain ribbon** ONCE: step along the route at `terrainSegLen`, sample `centerlineAt(s)` for the spine, lay a strip of width `2·ribbonHalfWidth` with `terrainSubdiv` lateral verts, each vert Y = `terrainHeight(route, z, x)`, normals from finite differences. One `BufferGeometry`, **zero per-frame allocation** (O17 identity check). PBR `MeshStandardMaterial` + procedural grass/rock/ballast textures, vertexColors grass→rock by slope. - **`src/render/scene.ts` (CHG).** (a) Camera/cab read `cameraFrame`; the live cab attitude **eases** toward `cabAttitudeTarget` via `approach()` (render-local). (d) **Viaduct** (instanced piers to the valley floor + arch spans + river plane) and **tunnel** (portals + dark bore) drawn where the predicates fire. Flat ground deleted.
- **`src/render/scenery.ts` (CHG).** Delete `addBank` and cone-hills (terrain owns relief now). Every prop's Y from `anchorY` — props sit on the real terrain. Trees/fences/masts/buildings/people keep their instanced one-mesh-per-class structure, re-anchored; prop detail picked by `lodForDistance`.
- **`src/render/textures.ts` (NEW).** Canvas-2D tileable grass/rock/ballast/water albedo + normal maps (value-noise → height → normal), built once. **No committed image assets.**
- **`src/render/cab.ts` (CHG, tiny).** Mount receives pitch+roll (eased) from scene; furniture unchanged.
- **`src/main.ts` (CHG, tiny).** Use `KINGSGATE_SEAHAVEN`; pass the extended quality opts; project the realism params; feed `cameraFrame`/look offset. No arithmetic added.

**Why the split is the point:** if a tree floats, a bank steps, the viaduct lands wrong, the curve doesn't bend, the cab over-rolls, or the ribbon blows its vertex budget — **a pure oracle fails on Node before any pixel is drawn.** The screenshot gate answers only what machines can't: *does it look like a railway threading real country?*

## 4. Acceptance criteria (numbered oracles — definition of done)

**Pure unit oracles (vitest, Node, no GPU):**
- **O1 — height integral (analytic).** Single grade `g` over `[0,L]` ⇒ `heightAt(s)===g·clamp(s,0,L)` (≤1e-9); multi-segment equals the hand-computed piecewise sum at every breakpoint & midpoint.
- **O2 — height ↔ grade (differential).** Central finite difference of `heightAt` equals `gradeAt` to 1e-6 everywhere ≥ ε from a breakpoint, sampled densely.
- **O3 — height C0.** `|heightAt(b−δ)−heightAt(b+δ)| ≤ 1e-9` at δ=1e-6 across every grade breakpoint.
- **O4 — tangent G1 (no kink).** Unit tangent from `centerlineAt` continuous (≤1e-6) across every curvature breakpoint; `‖tangent‖≈1` everywhere.
- **O5 — lateral offset (arc oracle).** Single constant-κ arc `R=1/κ`, `θ=κL` ⇒ `x=R(1−cosθ)`, advance `z=R sinθ` (≤1e-6). Straight ⇒ `x` constant, `z(s)=s`.
- **O6 — cant.** 0 on straights; on arcs `0<|cant|≤CANT_MAX`, sign matches turn; pure (same input → same output).
- **O7 — anchor invariant.** ∀ `(z,x,clearance≥0)`: `anchorY−terrainHeight===clearance` (≤1e-9) and `anchorY≥terrainHeight` — nothing floats/sinks.
- **O8 — terrain blend.** `|x|≤SHOULDER` ⇒ `terrainHeight===formation+ballastShoulder` (weight 0, ≤1e-9); far out the weight is 1; no NaN/Inf across the whole route.
- **O9 — predicates disjoint & deterministic.** `viaductSpanAt(s)` true on every authored viaduct chainage and false ≥50 m outside; same for `tunnelBoreAt`; `viaductSpanAt && tunnelBoreAt` never both true; `terrainHeight` bit-identical on repeat calls; changing `terrainSeed` changes the field (non-vacuous).
- **O10 — realism monotonicity.** `exposure` and `bloomStrength` finite and ordered as designed across `time×weather` (exposure day>dusk>night for fixed weather).

## 5. Residual risks
- **R1 — Aesthetic quality isn't machine-checkable.** Oracles prove geometric correctness (right place/height, bends, banks, spans); only a human confirms it *looks like* real country. Flagged for Arthur (S1–S4).
- **R2 — Perf on a 14 km route.** Build-once ribbon + tier gates (O13/O18) keep it bounded, but frame-rate is a *sanity check*, not an oracle; the implementer must spot-check interactive FPS desktop + mobile-emulated. The far field is a cheap static horizon band, not geometry.
- **R3 — Cab attitude comfort.** O16 bounds it (~4° roll, eased, 0 on mobile); "comfortable" is partly subjective — a single tested `attitudeScale`/clamp to tune, or 0 to fall back to yaw+pitch-only with the oracle still green.
- **R4 — `three/examples` import paths under Vite + `base "./"`.** EffectComposer/UnrealBloomPass are ESM sub-imports of installed `three`; if a path won't bundle, fall back to the gradient sky dome and skip bloom (already gated to 0 on mobile) — no new dep either way.
- **R5 — Value-noise tuning.** `naturalGround` must read as gentle rolling land, not jagged spikes — an octave/amplitude tuning loop. Determinism (O9) is guaranteed; pleasantness is screenshot-tuned.
- **R6 — z(s) < chainage on curves.** The eye's world-z trails the scalar chainage slightly on curves; harmless (render-only) but the implementer must drive the camera from `centerlineAt(chainage)`, never assume `z===chainage` (asserted indirectly by O5).
- **R7 — Tunnel interior is cheap.** Fog + portal light, not a modelled bore; if it reads as a black hole, upgrade to a short textured tube (render-only, no seam impact).
- **R8 — Nearest-chainage attribution for terrain `(z,x)`.** The blend needs `s*` per terrain vertex; the "monotone-in-z + small correction" closed form is exact only while curvature stays gentle (R ≥ 250 m, enforced by O19). A future sharper route would fail O19 loudly rather than mis-attribute silently. We deliberately do **not** build a general nearest-point search (the over-engineering we're cutting).

## 6. Decision log
- **D1 — The world is math; carve it all pure.** Centreline, terrain, anchor, camera are pure functions of route data under `src/sim`, oracle-tested, G3-guarded; Three.js becomes a dumb projector. *(Winner thesis: testability-first.)*
- **D2 — Curvature → lateral *offset* via closed-form arc, not a 2-D track plan.** Exact analytic oracles (O5), physics keeps 1-D chainage. A free 2-D bezier plan is the on-ramp-to-nowhere we refuse to build.
- **D3 — Terrain = blend(formation, natural) by |x|; one function for cuttings *and* embankments.** The sign of `natural − formation` does all the work; set-pieces are authored via route bands (O8/O9). *(Best single idea, from simplest-viable; shared by all three drafts.)*
- **D4 — Predicates over flags.** `viaductSpanAt`/`tunnelBoreAt` are pure thresholds on `reliefAt` (disjoint, O9) — the structure exists *because* the land demands it. No feature flags anywhere.
- **D5 — Gradient sky dome baseline (graft from simplest-viable D5).** A dome reading `environment.ts`'s palette honours the wet-night signature and costs one mesh; `three/examples` `Sky` (daytime Rayleigh) is *not* the baseline. PMREM still bakes off the dome for reflections on the strong tier.
- **D6 — Realism params pure; the *gates* live in quality.** environment says what's *wanted* (exposure/sunDir/bloom); quality says what the *device allows* (shadows/bloom on/off, ribbon detail, attitudeScale). Both pure, both tested. No env vars, one device-derived branch.
- **D7 — Cab rides pitch + *clamped, scaled, eased* roll (fork resolved toward realism, proved safe).** Pure `cabAttitudeTarget` (O16) bounds amplitude and is **0 on mobile/reduced-motion**; the render layer eases toward it via the existing `approach()`. We keep the signature viaduct bank *and* a machine-checked comfort bound. **Graft of risk-first's eased+scaled attitude over a hard clamp** (judge 3). *Dissent:* yaw-only is safer (current behaviour); rejected because it discards the centrepiece sensation on the route's headline set-piece, and the clamp is one tested dial (R3) settable to 0 to recover it.
- **D8 — Build-once track-following ribbon, NOT chunked streaming.** A ±180 m (desktop) / ±110 m (mobile) ribbon + a cheap static far-horizon band + fog covers everything the cab can see on a hand-authored 14 km line. **Rejected risk-first's chunked/skirts/seam-weld/LOD-hysteresis subsystem** as the speculative machinery the binding brief warns against. *(Tie-break; the 2 majority judges agree.)* Ribbon dimensions are quality-tier outputs (O13) + a numeric budget (O18), so "mobile must not break" is tested, not hoped.
- **D9 — Preserve/re-pin ENV4 for the Grand Route.** The default environment's μ keeps the longitudinal-physics tests green; if the Grand Route warrants a new default, move ENV4's pinned value with it in the same commit. Physics is otherwise untouched and forbidden from importing the 3-D path (O17).
- **D10 — Replace the route, don't keep two.** Dev sandbox: `KINGSGATE_SEAHAVEN` becomes default; `WESTFORD_EASTBANK` kept only if a test needs it. New per-route fields are optional so any retained route type-checks.
- **D11 — Centreline stays C0-position / G1-tangent; NO C1 transition (easement) curves.** **Rejected risk-first's VT_RAMP/TRANS_RAMP** machinery: it complicates the *exact* analytic oracles (O1/O5) and adds moving parts the brief's binding simplicity forbids. The **eased cab attitude (D7)** supplies *felt* C1 smoothness across breakpoints without geometry machinery. *(Lead tie-break; revisit only if a screenshot shows a visible kink — gentle authored grades/curves make this unlikely.)*
- **D12 — Numeric budgets + zero-alloc identity (graft from risk-first O12/O14/O16).** `ribbonVertexCount` (O18), the ≤400 KB-gz + no-image-asset build check (O22), and the BufferAttribute-identity zero-alloc assertion (O21) turn "mobile-reach" and "static-host" from screenshots into Node-side numbers. The one thing the winning draft genuinely lacked. *(All 3 judges flagged this.)*
- **D13 — Eye-clearance clamp + far-horizon band (graft from risk-first R8 / simplest R1).** `clampEye` (O20) stops a pitching camera ducking under the land; a cheap static far-horizon band + fog hides the ribbon's edge so distant skyline reads beyond the strip.
- **D14 — `test/reach.test.ts` is CHG.** Its QUAL assertion uses `toEqual` (exact object match), so adding any field to `QualitySettings` requires updating the expected object (existing field values unchanged). *(Judge 2's catch — a real break if missed.)*
