# HLD — mdtrain2 "Grand World": living terrain, a grand route, a push to realism — V3

**Status:** DRAFT (V3) · Date 2026-06-15 · Target `C:\language\mdtrain2` (Phases 0-5 merged, v0.10.0). `package.json` 0.10.0 → **0.11.0**.
**Provenance:** V1 = ultracode synthesis (testability-first winner + grafts). V2 = + review round 1 (25 findings). **V3 = + review round 2** (17 confirmed: 3 majors — the ACES/`OutputPass` wiring, the env-map "sheen" that was a no-op, and the viaduct/tunnel placement that must go through `placeOnCentreline`; plus pinned feature/LOD/eye constants and the ribbon-seam contract). Round 2 surfaced **0 criticals** ⇒ the adversarial review loop **exits** after V3. The design held; the fixes are localised.

The world is a flat green plane the rails lie *on*. This overhaul makes it a country the line **threads through** — cuttings that cut, embankments that carry, a river valley spanned by a **viaduct**, a headland bored by a **tunnel** — on a longer, seven-chapter **Grand Route**, lit with ACES tone-mapping, PBR materials, one real sun and a real sky. The governing instinct: **the world is geometry, and geometry is math.** Every metre of track, every terrain height, every prop's resting place, every "is this span a viaduct?" decision is a *pure function of route data in curvilinear coordinates*, computed and checked on Node with no GPU. Three.js becomes a thin, named, screenshot-verified projector that draws what the pure layer computed.

## 1. Scope & non-goals

### In scope
1. **A pure parametric centreline** (`src/sim/centerline.ts`, NEW, PURE, TESTED): `centerlineAt(route, s) → Frame{x,y,z, tangent, up, heading, cant}`; height = **analytic ∫grade**; planar position = **analytic ∫∫curvature** (per-segment arc displacement *rotated by the running heading ψ₀*); C0 position, G1 tangent. Plus `placeOnCentreline(route, s, d) → {x,y,z, yaw}` — **the single placement primitive for every lineside object *and* set-piece structure** — and the (folded-in) camera helpers `cameraFrame`, `cabAttitudeTarget`. **The prime seam.**
2. **A pure terrain model in curvilinear coordinates** (`src/sim/terrain.ts`, NEW, PURE, TESTED): `terrainHeight(route, s, d)` blends the rail **formation height** with deterministic seeded-value-noise **natural ground** by `|d|`. The signed difference `natural − formation` *is* the scenery; authored macro-relief uses a **flat-bottomed plateau + smoothstep shoulder** keyed to the viaduct/tunnel bands. Plus `reliefAt`, predicates, `anchorY`, `clampEye`. No world→chainage inversion.
3. **A real living landscape** rendered from the above: a **build-once, seam-welded, sectioned** track-following terrain ribbon, a **multi-arch viaduct** and a **tunnel** (both placed via `placeOnCentreline`+heading so they follow the bent spine), **sea/river** planes, and **every prop placed via `placeOnCentreline`+`anchorY`**. The flat ground, cone "hills" and faked `addBank` cuttings are **deleted**.
4. **A push toward realism, free-first, tier-gated:** `ACESFilmicToneMapping` set on the renderer for **both** the direct-to-canvas and the composer (`OutputPass`) paths so it is applied exactly once (D17); an sRGB-correct pipeline; a **palette re-tune** of the existing emissives (D14); **procedural canvas textures + normal maps** (grass/ballast/rock/water — *no committed image assets*); a **gradient sky dome** (`fog:false`) with a render-local zenith darkening; a **tiny (16×16) procedural gradient env map** on `scene.environment` for cheap IBL ambient + wet-rail sheen (D5'); and — **tier-gated** — **one** eye-tracking shadow-casting sun and an **optional** lazy-imported bloom pass. Everything scales down through `quality.ts`; nothing breaks if it's all off.
5. **A grand multi-chapter route** (`src/sim/route.ts`, CHG, data): `KINGSGATE_SEAHAVEN`, ~**14 km**, **7 chapters**, optional `viaducts[]`/`tunnels[]`/`terrainSeed`. `WESTFORD_EASTBANK` **retained as a test fixture**.

### Non-goals (ruthless cuts)
- No 2-D track plan / free bezier (D2). No physics change (physics keeps 1-D `gradeAt`; never imports the 3-D path — O17). No world→chainage inversion (curvilinear `(s,d)` everywhere). No chunked/streaming terrain (one build-once sectioned ribbon + far-horizon band + fog — D8). No C1 easements (eased cab attitude gives felt smoothness — D11). No full **scene-cube** PMREM, no water sim, no modelled tunnel interior, no god-rays, no day/night cloud sim. No new npm dependency. No env vars, no feature flags. No migration/back-compat except keeping `WESTFORD_EASTBANK` as a fixture.

## 2. The design

### 2.0 Architecture in one breath
```
route data (1-D bands: grades, curvatures, speedLimits, signals, + optional viaducts/tunnels/terrainSeed)
   │
   ├─ physics.ts ──────────── gradeAt(route,s)                              [UNCHANGED 1-D path]
   │
   └─ centerline.ts (PRIME SEAM) ── centerlineAt(route,s) → Frame{x,y,z,tangent,up,heading,cant}
        ├─ placeOnCentreline(route,s,d) → {x,y,z,yaw}     ← single placement primitive for ALL props AND set-pieces
        ├─ cameraFrame(route,s,look,eye) → {position,yaw,pitch,roll}
        ├─ cabAttitudeTarget(cant,grade,scale) → {pitch,roll}
        └─ terrain.ts ── terrainHeight(route,s,d); reliefAt(route,s); viaductSpanAt; tunnelBoreAt;
                          anchorY(route,s,d,clearance); clampEye(route,s,eyeD,eyeY)

environment.ts ── environmentParams(env) → {…, exposure, sunDir, sunColorPbr, bloomStrength}
quality.ts     ── qualityFor(env) → {…, shadowsEnabled, bloomEnabled, ribbonHalfWidth, terrainSegLen,
                                       terrainSubdiv, attitudeScale}; lodForDistance(d); ribbonVertexCount(tier,len)

render/* (IMPURE, screenshot-only): read the pure layer, ease the cab attitude, place & draw meshes. No spatial math.
```
Everything left of `render/*` is pure, lives under `src/sim`, contains **no banned token** (`Date|Math.random|performance.now|setTimeout|setInterval`; integer-hash noise; `Math.sin/cos/sqrt/atan` allowed), and is oracle-tested. The **G3 grep glob is widened** to also cover `src/render/quality.ts` (D6'), so "G3 polices the whole spatial model" is literally true.

### 2.1 `src/sim/centerline.ts` (NEW, PURE, TESTED) — the prime seam
```ts
export interface Frame { x:number; y:number; z:number; tangent:Vec3; up:Vec3; heading:number; cant:number; }
export function centerlineAt(route: Route, s: number): Frame;
export function heightAt(route: Route, s: number): number;          // ∫₀ˢ grade (analytic, piecewise-linear ⇒ C0)
export function lateralOffsetAt(route: Route, s: number): number;   // ∫∫ curvature (analytic)
export function headingAt(route: Route, s: number): number;          // tangent yaw, rad
export function placeOnCentreline(route: Route, s: number, d: number): { x:number; y:number; z:number; yaw:number };
export function cameraFrame(route: Route, s: number, look:{yaw:number;pitch:number}, eye: EyeOffset): CamFrame;
export function cabAttitudeTarget(cant: number, grade: number, scale: number): { pitch:number; roll:number };
```
- **Height** = `Σ grade_i·(seg overlap with [0,s])`, clamped `[0,length]`; piecewise-constant integrand ⇒ piecewise-linear integral ⇒ C0 (O1–O3).
- **Planar position:** heading `ψ(s)=ψ₀+∫κ`. **Per segment, compute the LOCAL arc displacement and ROTATE it by the entry heading ψ₀ before adding to the running pose** — for a constant-κ arc length `L`, `Δworld = Rot(ψ₀)·(R(1−cosθ), R sinθ)`, i.e. `x += (cosψ₀ − cos(ψ₀+θ))/κ`, `z += (sin(ψ₀+θ) − sinψ₀)/κ`, `θ=κL, R=1/κ`; straight advances `L` along `ψ₀`. Accumulate `(x,z,ψ)`; `ψ` carries across so the tangent is G1 though κ steps (O4, O5 single-arc, **O5b two-arc compounded pose**).
- **cant** = `clamp(CANT_GAIN·|κ(s)|·v_ref(s)², CANT_MAX)`, signed, `v_ref = speedLimitAt(s)`. **Pinned `CANT_GAIN=0.08`, `CANT_MAX=0.105 rad (6°)`** (O6, non-tautological).
- **`placeOnCentreline(route,s,d)`:** `f=centerlineAt(s)`; world pos `=(f.x + d·rightX, f.y, f.z + d·rightZ)`, `right=(cos f.heading, −sin f.heading)` the in-plane normal; `yaw=f.heading`. **Every signal head, station, mast, fence, tree, milepost, OLE wire point, person, AWS marker, the rail/sleeper sweep, AND the viaduct piers/arches + tunnel portals/bore** are placed through this — the whole world follows the curve and faces down the track (O5c).
- **`cameraFrame` / `cabAttitudeTarget`** (folded from the abandoned `camera.ts` — there is **no** inline trig in today's `scene.ts` to extract, D1'): `cameraFrame` returns `position = clampEye(route, s, EYE_D, place.y + EYE_HEIGHT)` over `place=placeOnCentreline(s, EYE_D)`; `yaw=heading+look.yaw`; `pitch=atan(grade)+look.pitch`; `roll` from `cabAttitudeTarget`. `cabAttitudeTarget(cant,grade,scale)={ pitch:clamp(scale·atan(grade), ±PITCH_MAX), roll:clamp(scale·cant, ±ROLL_MAX) }`. **Pinned `EYE_HEIGHT=1.9`, `EYE_D=−0.5` (driver's offset), `ROLL_MAX=0.070 rad (4°)`, `PITCH_MAX=0.052 rad (3°)`, `attitudeScale=0.35` (desktop)/`0` (coarse & reduced-motion).** *Note (O16, D7):* in the live pipeline `cant` is already clamped to `CANT_MAX` and grades are gentle (≤~1.2 %), so `scale·CANT_MAX≈0.037 < ROLL_MAX` — **`ROLL_MAX`/`PITCH_MAX` are defensive ceilings for arbitrary inputs; O16 exercises them with synthetic over-the-clamp values.** The **ease** (advancing the live attitude toward the target via `approach()`) is render-local state in `scene.ts`.
- `Math.sin/cos/sqrt/atan` allowed; no clock. Honest planar `z(s) ≤ s` on curves; physics never reads it (R6).

### 2.2 `src/sim/terrain.ts` (NEW, PURE, TESTED) — the living land, curvilinear
```ts
export function formationHeight(route: Route, s: number): number;          // = heightAt + ballast shoulder near track
export function naturalGround(route: Route, s: number, d: number): number; // seeded value noise + authored relief
export function terrainHeight(route: Route, s: number, d: number): number; // blend(formation, natural) by |d|
export function reliefAt(route: Route, s: number): number;                 // natural(s,0) − formation(s)
export function viaductSpanAt(route: Route, s: number): boolean;           // reliefAt < −VIADUCT_THRESH
export function tunnelBoreAt(route: Route, s: number): boolean;            // reliefAt >  TUNNEL_THRESH
export function anchorY(route: Route, s: number, d: number, clearance: number): number; // terrainHeight + clearance
export function clampEye(route: Route, s: number, eyeD: number, eyeY: number): number;  // ≥ terrainHeight(s,eyeD)+MIN_EYE_CLEARANCE
```
**Curvilinear `(s,d)` throughout** — the fix that makes the blend, ballast corridor, formation height and feature attribution exact on curves. The ribbon walks `s` and lays verts at offset `d`, so it has `(s,d)` directly.

**The blend:**
```
formation(s)       = heightAt(route, s)
natural(s,d)       = baseProfile(route, s, d) + valueNoise(seed, s, d)
w(d)               = smoothstep(0, BLEND_W, |d| − SHOULDER)
terrainHeight(s,d) = lerp( formation(s) + ballastShoulder(|d|), natural(s,d), w(d) )
```
**Pinned:** `SHOULDER=4 m`, `BLEND_W=80 m`, `BALLAST_DROP=0.4 m`, `VIADUCT_THRESH=18 m`, `TUNNEL_THRESH=12 m`.
- `|d|≤SHOULDER` ⇒ `w=0`, terrain **is** formation+ballast (clean bed hugging the curved track, O8). `|d|≥SHOULDER+BLEND_W` ⇒ natural land. Between, `smoothstep`: `natural>formation` ⇒ **cutting**; `natural<formation` ⇒ **embankment**.

**`naturalGround` (deterministic, banned-token-free):** seeded integer-lattice value noise (mulberry-style hash on `terrainSeed + (⌊s/CELL⌋,⌊d/CELL⌋)`, bilinear, 2–3 octaves) for rolling texture, **plus authored macro-relief as a flat-bottomed PLATEAU + smoothstep shoulder** (D18): for each `viaducts[].center±halfLen` a trough of depth `valleyDepth` with a `SHOULDER_RAMP`-wide smoothstep edge; symmetrically for `tunnels[]` (a `hillHeight` plateau). **Pinned `SHOULDER_RAMP=120 m`.** So `reliefAt ≈ −valleyDepth` (or `+hillHeight`) flat across the band, and decays below `VIADUCT_THRESH`/`TUNNEL_THRESH` **within `SHOULDER_RAMP` of the band edge** — making O9's "predicate false outside the band" true by construction. Authored relief is intentionally **bound to the viaduct/tunnel bands** (one band = one feature + its relief); independent rolling relief comes free from the value noise (D18).

**Predicates** are pure thresholds on `reliefAt` (disjoint, O9). **`anchorY`** = tested invariant (O7). **`clampEye`** floors the eye at `terrainHeight(s,eyeD)+MIN_EYE_CLEARANCE` (`MIN_EYE_CLEARANCE=0.5 m`) (O20).

### 2.6 Render adapters (IMPURE, screenshot-verified) — no spatial math
- **`src/render/terrain-mesh.ts` (NEW).** Build the **track-following ribbon** ONCE on a single unified `(s,d)` sample grid (step `s` at `terrainSegLen`; lateral verts at offsets `d∈[−ribbonHalfWidth,+ribbonHalfWidth]`; world pos `placeOnCentreline(s,d)`, Y `terrainHeight(s,d)`). **Seam contract (finding #10):** compute vertex normals by **central differences on the unified grid FIRST**, *then* slice the grid into **`RIBBON_SECTIONS = ceil(length/SECTION_LEN)` (pinned `SECTION_LEN=900 m` ⇒ ~16)** along-track sections that **share boundary vertices — positions AND normals copied identically** — so the sections cull independently with no seam/normal popping. - **`src/render/scene.ts` (CHG).**
  - **(a) Camera/cab:** read `cameraFrame`; the impure layer sets `camera.rotation.set(easedCabPitch + look.pitch, heading + look.yaw, easedRoll)` (YXZ), where `easedCabPitch/easedRoll` `approach()` toward `cabAttitudeTarget`; `cabMount` takes the same eased pitch/roll (no look offset).

No-composer (mobile) path: the canvas render applies ACES+sRGB. (This is a *tiny* procedural env map auto-PMREM'd by the renderer, **not** a full scene-cube PMREM — D5'.)
  - **(f) Shadows:** one **eye-tracking** shadow-casting `DirectionalLight` along `env.sunDir`: a **±100 m ortho frustum moved with the eye each frame**, `2048²` map (desktop), tuned `bias`/`normalBias`, tight near/far; gated by `shadowsEnabled` (finding #12).

Flat ground deleted.

- **`src/render/textures.ts` (NEW).** Canvas-2D tileable albedo+normal maps + the tiny env-map gradient. **No committed image assets.**
- **`src/render/cab.ts` (CHG, tiny).** Mount takes eased pitch+roll.
- **`src/main.ts` (CHG, tiny).** Grand Route; quality opts; realism params; camera frame.

## 4. Acceptance criteria (numbered oracles — definition of done)

**Pure unit oracles (vitest, Node, no GPU):**
- **O1** height integral (analytic) ≤1e-9; **O2** height↔grade differential ≤1e-6 off breakpoints; **O3** height C0 ≤1e-9 at every breakpoint.
- **O4** tangent G1 ≤1e-6 across every curvature breakpoint; `‖tangent‖≈1`.
