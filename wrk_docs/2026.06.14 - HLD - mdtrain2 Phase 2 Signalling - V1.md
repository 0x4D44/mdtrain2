## 2. The design

### 2.0 Shape, derived from the oracles (testability-first)

The oracle list (Â§4) demands exactly **four pure functions** plus **one threaded state record**. The design is shaped to *that* and nothing more:

| Oracle group | Function under test | Inputs it forces |

### 2.1 Route signal/AWS data model (`route.ts`)

Signals are **point features**, not piecewise-constant fields â€” they get their own sorted array and a point query. **Do not** force them into `Segment<T>` (that abstraction is `[from, to)` intervals; misusing it is the M8-on-ramp trap). Aspects are **derived**, never stored.

```ts
// route.ts â€” NEW
export type Aspect = "RED" | "YELLOW" | "DOUBLE_YELLOW" | "GREEN";

export interface Signal {
  /** Signal-post / stop-line chainage, m. The single stored chainage. */
  chainage: number;
  /** Station whose unserved booked stop holds this signal RED ("" = plain
   *  block signal, never self-held; red only via the ahead cascade). */
  protects: string;
}

export interface Route {
  length: number;
  stations: Station[];
  grades: Segment<number>[];
  speedLimits: Segment<number>[];
  curvatures: Segment<number>[];
  signals: Signal[]; // NEW â€” required; sorted ASCENDING by chainage (authoring invariant)
}
```

- **One chainage per signal.** The AWS magnet (`chainage âˆ’ AWS_MAGNET_OFFSET`) and the OSS loop (`chainage âˆ’ OSS_LOOP_OFFSET`) are **derived** at use-site, never stored. This is the leanest believable shape (mirrors legacy `sAws = max(0, sâˆ’100)`). Constants live in `aws.ts`; **`STAND_EPS` is not overloaded**.
- `signals` is **required** (house style: required field + explicit default, cf. `SimInputs.emergency`). The three synthetic fixtures get `signals: []`, which keeps every Phase-0/1 oracle green and makes the O13 migration trivial (empty signals â‡’ `tickAws` returns no reasons).
- **No `blockSections`.** The served-stop proxy needs none; adding interval block data now would be premature (D7).

Two new pure point queries, in `route.ts`, mirroring `lookup`'s guarded style:

```ts

/** Indices of signals whose `chainage` lies in the forward swept interval (sPrev, sNow]. */

```

Direction reconciliation lives in `tickAws` (Â§2.4), so `signalsCrossedFwd` only handles the forward case (`sNow > sPrev`); reverse/standing crosses nothing. Both are end-clamp-safe and array-index-guarded for `noUncheckedIndexedAccess`.

### 2.2 Aspect rule (pure, train-free)

A signal `i` is **held RED** iff its `protects` station is **not in `served`** â€” the single-player "section occupied" proxy. Otherwise it cascades off **one** signal ahead. `served` lives in `AwsState` (Â§2.3).

```ts
// route.ts â€” pure, O(1) per signal, no train, no dt
export function aspectAt(route: Route, i: number, served: ReadonlySet<string>): Aspect {
  const sig = route.signals[i];
  if (!sig) return "GREEN";                                   // guard (noUncheckedIndexedAccess)
  if (sig.protects !== "" && !served.has(sig.protects)) return "RED"; // section occupied
  const ahead = route.signals[i + 1];                         // next signal toward Eastbank
  if (!ahead) return "GREEN";                                 // last signal, nothing ahead
  const aheadAspect = aspectAt(route, i + 1, served);         // backward sweep, depth â‰¤ n
  if (aheadAspect === "RED") return "YELLOW";
  if (aheadAspect === "YELLOW") return "DOUBLE_YELLOW";
  return "GREEN";
}
```

Recursion depth is bounded by `signals.length` (4); index strictly increases â‡’ no cycles; terminates at the last signal (oracle A4 pins the guarded out-of-range read). Double-yellow is reachable and meaningful: with only S4 unserved, S4=RED, S3=YELLOW, S2=DOUBLE_YELLOW, S1=GREEN.

**Served-set writer.** `served` grows in the `tickAws` path: a station enters `served` the first tick the train front is within `Â±platformHalf` of its stop board **and** `|speed| â‰¤ STAND_EPS`. This is the only writer; it is a position-only proxy, not a dwell model (Residual #2).

### 2.3 `AwsState` + the new `tickAws` signature and state machine

Per the brief's literal mandated signature, `prevChainage` is passed **explicitly** (not folded into `AwsState`), so there is a single source of truth for the previous front â€” the impure shell (`main.ts`) owns it; `tickAws` stays pure on its explicit argument.

```ts
// aws.ts â€” owned here, threaded by main.ts; pure, dt-counted
export type AwsPhase = "CLEAR" | "WARNING" | "BRAKE";
export type Sunflower = "BLACK" | "CAUTION"; // all-black = clear; black/yellow = caution-latched

export interface AwsState {
  phase: AwsPhase;        // AWS warning/brake machine
  warnTimer: number;      // s remaining in the ack window; counts DOWN off dt (WARNING only)
  sunflower: Sunflower;   // latched caution reminder; reset only by a GREEN magnet
  brakeReason: PenaltyReason | null; // single BRAKE-latch source: "AWS" | "TPWS" | null
  served: ReadonlySet<string>;       // stations whose booked stop is done (occupancy proxy)
  spad: boolean;          // sticky SPAD event flag (HUD/scoring)
}

export interface AwsHud { aspect: Aspect; sunflower: Sunflower; spad: boolean; }
export interface AwsOutput { reasons: PenaltyReason[]; } // UNCHANGED latch-boundary type

export function createInitialAws(): AwsState {
  return { phase: "CLEAR", warnTimer: 0, sunflower: "BLACK",
           brakeReason: null, served: new Set(), spad: false };
}

export function tickAws(
  aws: AwsState,
  state: SimState,
  route: Route,
  intent: ControlIntent,
  prevChainage: number,
  dt: number,
): { next: AwsState; reasons: PenaltyReason[]; hud: AwsHud };
```

**One BRAKE latch path for AWS + TSS + OSS.** Rather than two booleans, a single `phase === "BRAKE"` plus a one-field `brakeReason: "AWS" | "TPWS" | null` carries the latched demand. Both an AWS-deadline miss and a TPWS TSS/OSS trip set `phase = "BRAKE"` and `brakeReason`; both re-assert *their own* reason every BRAKE tick and both clear identically on ack-at-stand. Fewer fields, one persistence mechanism, one clear transition (D5).

**`tickAws` owns its AWS/TPWS latch lifecycle.** It returns the reason **every tick while latched** (BRAKE), and **stops returning it the tick its own internal latch clears** (AWS: the ack-at-stand release transition; TPWS: stop + reset). Because the `tickSafety` seed is now DSD-scoped (Â§2.4), the moment `tickAws` stops asserting a reason, that reason is no longer in the set the next tick. The latch lifecycle is fully owned by `tickAws`; `tickSafety` merely folds whatever `tickAws` returns this tick.

#### The state machine (one tick, all `dt`-counted)

Front position uses a fixed `FRONT_OFFSET` (legacy 2.0 m). `dir = sign(state.speed)`; the swept interval is `(frontPrev, frontNow]` with `frontNow = state.chainage + FRONT_OFFSET`, `frontPrev = prevChainage + FRONT_OFFSET`. **All edge logic fires only on strict forward motion `frontNow > frontPrev`** â€” reverse or standing crosses nothing (D10), so a reversing train never false-triggers AWS/TPWS/SPAD.

1. **Served update**: if `|speed| â‰¤ STAND_EPS` and front within `Â±platformHalf` of a station, add it to `served` (copy-on-write). Releases protecting signals ahead.
2. **Magnet crossings** (`signalsCrossedFwd` over each signal's derived magnet chainage): for each magnet crossed, evaluate that signal's `aspectAt`:
   - **GREEN magnet** â†’ `sunflower = "BLACK"`; if `phase === "WARNING"`, cancel to `CLEAR` (real-AWS clear bell). Does **not** clear a `BRAKE`.
   - **non-GREEN magnet** â†’ `phase = "WARNING"`, `warnTimer = AWS_WARN_WINDOW`, `sunflower = "CAUTION"`.
3. **WARNING countdown**: if `phase === "WARNING"`:
   - `intent.acknowledge` (allowed **while moving** â€” D4) â‡’ `phase = "CLEAR"`, sunflower stays `CAUTION` (latched reminder), no penalty.
   - else `warnTimer âˆ’= dt`; if `warnTimer â‰¤ 0` â‡’ `phase = "BRAKE"`, `brakeReason = "AWS"`.
4. **TPWS / SPAD** (`signalsCrossedFwd` over signal chainages, evaluating `aspectAt` at the crossing):
   - **TSS / SPAD**: signal crossed while `aspect === "RED"` â‡’ `phase = "BRAKE"`, `brakeReason = "TPWS"`, `spad = true` (the train-stop trip *is* the SPAD).
   - **OSS**: the signal's OSS loop crossed while that signal `aspect === "RED"` **and** `|speed| > OSS_TRIP_SPEED` â‡’ `phase = "BRAKE"`, `brakeReason = "TPWS"`.
5. **BRAKE persistence / release**: while `phase === "BRAKE"`, the machine re-asserts `brakeReason` every tick. Release: `intent.acknowledge && |speed| â‰¤ STAND_EPS` â‡’ `phase = "CLEAR"`, `brakeReason = null`, sunflower stays `CAUTION` until the next GREEN magnet. (`spad` stays sticky â€” a scored event.) The release tick flips `phase â†’ CLEAR` and sets `brakeReason â†’ null`, so the *next* tick `tickAws` returns `[]` and the reason is no longer carried by the DSD-scoped seed.
6. **Reasons assembled**: `reasons = brakeReason ? [brakeReason] : []`.
7. **HUD**: `aspect = nextSignalAhead(...)` projected through `aspectAt`, else `"GREEN"`; return `{ aspect, sunflower, spad }`.

All countdowns are `dt`-subtractive and all crossings use swept intervals, so coarse and fine `dt` reach equal end-states (G oracles). A coarse `dt` that jumps a magnet *and* its signal in one tick is handled because step 2 (magnet) and step 4 (signal) consult the same swept interval independently: enter WARNING, decrement past zero, fall through to BRAKE within the one tick.

**Ack split (D4).** The WARNING-window ack (cancel-before-penalty) is internal to `tickAws` and allowed while moving; the latch-release ack (clearing an *asserted* reason) stays ack-at-stand. Both consume the single `intent.acknowledge` edge â€” no hidden second ack path.

### 2.4 How reasons re-assert to survive ack, and when they stop (load-bearing)

This is the keystone mechanism. It depends on **two** facts in `tickSafety`: the carry-forward seed at `controls.ts:258` is **scoped to DSD-only**, and the fold at `:275` runs **before** the DSD-only delete at `:278-281`. The DSD-scoping is the **enabling change (P2-RA)** without which the AWS/TPWS latch is permanently un-releasable.

**The seed scoping (the one-line edit at `:258`):**

```ts
const reasons = new Set<PenaltyReason>();
if (s.penaltyReasons.has("DSD")) reasons.add("DSD");
```

This replaces the original `const reasons = new Set<PenaltyReason>(s.penaltyReasons);`. After this edit, AWS/TPWS reasons are **never seed-carried** â€” they are re-derived **fresh from the fold** (`for (const r of aws.reasons) reasons.add(r)`, `:275`) on every tick. They are present in the set **iff `tickAws` returns them this tick**.

- **Survives ack (C1).** While `phase === "BRAKE"`, `tickAws` re-asserts `brakeReason` **every tick**; the fold (`:275`) re-adds it. The DSD-only delete at `:278-281` cannot touch it. So an ack-at-stand that empties DSD does **not** strand-clear the AWS/TPWS reason while `tickAws` still demands it â€” the fold re-adds it the same tick. This is exactly the pre-authorised "source still demands it" clause (`controls.ts:244-246`).
- **Stops / clears cleanly (C2).** The reason drops the first tick `tickAws` **stops** returning it. The BRAKE-release transition inside `tickAws` (`intent.acknowledge && atStand`) flips `phase â†’ CLEAR`, `brakeReason â†’ null`; the **next** tick `tickAws` returns `[]`. Now â€” **because the seed is DSD-scoped** â€” the reason is no longer seed-carried, the fold adds nothing, and the set empties â†’ the latch releases. **This requires the `:258` seed-scoping**: under the original full carry-forward seed, the reason would be re-seeded every subsequent tick from `s.penaltyReasons` and never deleted (the release names only "DSD"), making the latch un-releasable forever. Scoping the seed to DSD is what makes "`tickAws` stopped asserting" translate into "reason gone next tick."
- **Why the release machine stays untouched.** The original `:280` comment ("Phase-2 persisting reasons would re-add themselves above") assumed fold-sourced reasons are **not** seed-carried â€” i.e. that a reason present in the set comes *only* from this tick's fold, not from yesterday's seed. The seed-scoping is precisely what makes that assumption true. With the seed DSD-scoped, the release machine's "delete DSD only" rule is already correct: every other reason in the set is, by construction, this-tick fold output, and drops automatically when the fold stops producing it. So the release machine (`:277-281`), `resolveInputs`, and `reduceControls` need **no edit** â€” only the seed is scoped.
- **`tickAws` only ever touches its own reasons** â€” never `"DSD"` (R1), and DSD logic never touches `"AWS"`/`"TPWS"`.

> **`tickSafety` release machine + signature unchanged; the carry-forward seed scoped to DSD-only (one line, `:258`).** The seed-scoping is the single body edit; it is what makes AWS/TPWS releasable. Survive = the fold re-adds the reason after the DSD-only seed + DSD delete, every tick `tickAws` demands it; clear = `tickAws` stops demanding it **and** the DSD-scoped seed no longer carries it â‡’ it drops next tick.

## 4. Acceptance criteria / test oracles (definition of done)

All `environment:node`, no DOM, no wall-clock. Synthetic routes built inline; helpers thread `AwsState` in the existing idiom. These are the contract â€” done when they pass and nothing else regresses.

**A â€” aspect cascade** (`aspectAt`, train-free; `aspectRoute` = the 4 signals of Â§2.1):
- **A1**: `served = âˆ…` â‡’ S4=`RED`, S3=`YELLOW`, S2=`DOUBLE_YELLOW`, S1=`GREEN` (the documented cascade; double-yellow reachable).
- **A2**: every station served â‡’ all signals `GREEN`.
- **A3**: only S2's station unserved â‡’ S2=`RED`, S1=`YELLOW`, rest `GREEN` (single-step caution).
- **A4**: last signal with nothing ahead returns only its own `RED`/`GREEN` â€” no out-of-range read (`noUncheckedIndexedAccess` guard + recursion termination exercised).
- **A-EMPTY**: `aspectAt`/`nextSignalAhead` on a `signals:[]` route never throw; `nextSignalAhead â†’ null` â‡’ HUD `"GREEN"`.

**B â€” AWS warning machine** (`awsFor`; magnet at 1200, S1 held RED):

## 5. Residual risks

1. **Fold-before-delete order + seed-scoping coupling (highest).** The persistence-and-release mechanism rests on two coupled facts: the seed at `:258` is **DSD-scoped**, and the fold (`:275`) runs **before** the DSD-only delete (`:279`). The **un-releasable-latch risk is now resolved by the seed-scoping** (not by ack-coincidence): once `tickAws` stops demanding a reason, the DSD-scoped seed drops it next tick regardless of any ack timing. What remains fragile is **fold-order**: if anyone moved the fold *after* the delete, or generalised the delete to remove non-DSD reasons, AWS/TPWS would mis-clear mid-demand. C1/C2/C2-PIN are the guards; their comments must cite the line numbers (`:258`, `:275`, `:279`).
2. **Served-set as occupancy proxy is a fiction.** Aspect colour couples to "drove well enough to stop." It is the simplest pure, testable single-player source; pinned by A2/A3 so the behaviour is at least *defined*. A train crawling through a platform without truly stopping could mis-flag. If it feels gamey, swap `served` for a scripted held-signal list with zero cascade changes.
3. **Double-yellow only on the S1â†’S2 section.** On a 6 km route a driver rarely sees two reds ahead. A1 proves the logic; the route's *use* of YY is decorative-but-correct. Re-spacing that closes the gap would lose the 4th aspect in play (logic still green).
4. **OSS abstraction.** Speed-threshold-at-a-loop-chainage, not real loop-spacing/arm-timer physics. Honest abstraction, flagged in code; trip speed is one tunable constant.
5. **Direction reconciliation edge cases.** `sign(speed)` near zero / oscillation at a stand. Mitigated by gating all crossings on strict `frontNow > frontPrev` (standing â‡’ no crossing); D2/G2 guard the reverse and coarse-dt cases â€” the subtlest geometry in the build.
6. **`prevChainage` plumbing is in the impure shell.** A wiring bug (passing post-step chainage as prev) is untestable at `main.ts`. Mitigation: `tickAws` is pure on its explicit `prevChainage`, so the *logic* is fully tested; only the one-line plumbing is trust-based (same risk class as the existing `dt` plumbing). B2/C1 span multiple ticks and catch a dropped `AwsState` thread.
7. **AWS magnet (200 m) vs short approaches.** S1/S3 magnets sit just before the 25 mph approach segments; signal placement is an authoring task to check against A1/A3, else the warning geometry is off. Tunable via `AWS_MAGNET_OFFSET`/placement; no logic change.
