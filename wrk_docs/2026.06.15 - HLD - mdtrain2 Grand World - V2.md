# HLD — mdtrain2 "Grand World": living terrain, a grand route, a push to realism — V2

**Status:** DRAFT (V2) · Date 2026-06-15 · Target `C:\language\mdtrain2` (Phases 0-5 merged, v0.10.0). `package.json` 0.10.0 → **0.11.0**.
**Provenance:** V1 = ultracode synthesis (testability-first winner + grafts). **V2 = V1 + adversarial-review round 1** (5 lenses incl. a devil's-advocate seat, 2-of-3 verification; 25 confirmed findings applied, 9 dropped). The dominant fix — **reparametrise the whole spatial model into curvilinear `(s, d)` coordinates** (chainage, signed lateral offset) — resolves eight findings at once and *deletes* V1's world→chainage inversion (former R8). PMREM deferred; `camera.ts` folded into `centerline.ts`; all constants pinned; the realism render-path made explicit (OutputPass, tracking shadow, ACES re-tune). The result is **simpler than V1**.

The world is a flat green plane the rails lie *on*. This overhaul makes it a country the line **threads through** — cuttings that cut, embankments that carry, a river valley spanned by a **viaduct**, a headland bored by a **tunnel** — on a longer, seven-chapter **Grand Route**, lit with ACES tone-mapping, PBR materials, one real sun and a real sky. The governing instinct: **the world is geometry, and geometry is math.** Every metre of track, every terrain height, every prop's resting place, every "is this span a viaduct?" decision is a *pure function of route data in curvilinear coordinates*, computed and checked on Node with no GPU. Three.js becomes a thin, named, screenshot-verified projector that draws what the pure layer computed.

## 1. Scope & non-goals

### In scope
1. **A pure parametric centreline** (`src/sim/centerline.ts`, NEW, PURE, TESTED): `centerlineAt(route, s) → Frame{x,y,z, tangent, up, cant, heading}`; height = **analytic ∫grade**; planar position = **analytic ∫∫curvature** (per-segment arc displacement *rotated by the running heading*); C0 position, G1 tangent. Plus `placeOnCentreline(route, s, d) → {x,y,z, yaw}` (lateral offset `d` from the line, with heading) — **the single placement primitive for every lineside object** — and the (folded-in) camera helpers `cameraFrame` and `cabAttitudeTarget`. **The prime seam.**
2. **A pure terrain model in curvilinear coordinates** (`src/sim/terrain.ts`, NEW, PURE, TESTED): `terrainHeight(route, s, d)` blends the rail **formation height** `heightAt(route,s)` (+ ballast shoulder near the track) with deterministic seeded-value-noise **natural ground** by `|d|`. The signed difference `natural − formation` *is* the scenery. Plus `reliefAt(route,s)`, `viaductSpanAt`/`tunnelBoreAt`, the prop-anchor `anchorY(route,s,d,clearance)`, and the eye floor `clampEye`. **Because everything is keyed on `(s, d)` there is no world→chainage inversion** (V1's R8 is gone).
3. **A real living landscape** rendered from the above: a **build-once track-following terrain ribbon** (sampled at `(s, d)` along the bent spine), a **multi-arch viaduct**, a **tunnel** bore + portals, **sea/river** planes, and **every prop placed via `placeOnCentreline` + `anchorY`** so the lineside follows the curve. The flat ground, cone "hills" and faked `addBank` cuttings in `scenery.ts` are **deleted**.
4. **A push toward realism, free-first, tier-gated:** `ACESFilmicToneMapping` via an explicit composer `OutputPass` (and a **palette re-tune** of the existing emissives — D14), an sRGB-correct pipeline, **procedural canvas textures + normal maps** (grass/ballast/rock/water — *no committed image assets*), a **gradient sky dome** (`fog:false`) reading the env palette, and — **tier-gated** — **one** eye-tracking shadow-casting sun and an **optional** bloom pass via `three/examples`. **PMREM is deferred** (D5'): wet rails get a cheap env-tinted sheen. Everything scales down through `quality.ts`; nothing breaks if it's all off.
5. **A grand multi-chapter route** (`src/sim/route.ts`, CHG, data): `KINGSGATE_SEAHAVEN`, ~**14 km**, **7 chapters** (city → suburb → countryside → viaduct → coast → tunnel → sea terminus), with optional per-route `viaducts[]`/`tunnels[]`/`terrainSeed` bands the pure terrain layer reads. `WESTFORD_EASTBANK` is **retained as a test fixture** (the signalling/physics tests reference it) but is no longer the live route.

### Non-goals (ruthless cuts)
- **No 2-D track plan / free bezier.** Curvature integrates to a planar path (closed form); that's all the eye needs. (D2)
- **No physics change.** `physics.ts`/`simulation.ts` keep `gradeAt` on the 1-D scalar chainage; the 3-D path is render-only and **physics never imports it** (O17, import-statement-scoped). 164 tests untouched.
- **No world→chainage inversion / nearest-point search.** Curvilinear `(s, d)` everywhere; the ribbon builder walks `s` and lays verts at offset `d` directly. (Dissolves V1's R8.)
- **No chunked/streaming/quadtree terrain.** One build-once ribbon + a cheap static far-horizon band + fog. (D8)
- **No C1 transition (easement) curves.** C0 position / G1 tangent; the eased cab attitude supplies *felt* smoothness. (D11)
- **No PMREM, no water sim, no modelled tunnel interior, no god-rays, no day/night cloud sim.** (D5')
- **No new npm dependency** (`three/examples/jsm/*` ship inside `three@0.183`). **No env vars, no feature flags** — device → the pure quality tier is the only branch.
- No migration / back-compat (dev sandbox), except keeping `WESTFORD_EASTBANK` as a test fixture.

## 2. The design

### 2.0 Architecture in one breath
```
route data (1-D bands: grades, curvatures, speedLimits, signals, + optional viaducts/tunnels/terrainSeed)
   │
   ├─ physics.ts ──────────── gradeAt(route,s)                              [UNCHANGED 1-D path]
   │
   └─ centerline.ts (PRIME SEAM) ── centerlineAt(route,s) → Frame{x,y,z,tangent,up,cant,heading}
        ├─ placeOnCentreline(route,s,d) → {x,y,z,yaw}     ← single placement primitive for ALL lineside objects
        ├─ cameraFrame(route,s,look,eye) → {position,yaw,pitch,roll}   (folded in; no separate camera.ts)
        ├─ cabAttitudeTarget(cant,grade,scale) → {pitch,roll}          (pure clamp; eased in render)
        └─ terrain.ts ── terrainHeight(route,s,d); reliefAt(route,s); viaductSpanAt; tunnelBoreAt;
                          anchorY(route,s,d,clearance); clampEye(route,s,eyeOffset)

environment.ts ── environmentParams(env) → {…, exposure, sunDir, sunColorPbr, bloomStrength}
quality.ts     ── qualityFor(env) → {…, shadowsEnabled, bloomEnabled, ribbonHalfWidth, terrainSegLen,
                                       terrainSubdiv, attitudeScale}; lodForDistance(d,tier); ribbonVertexCount(tier,len)

render/* (IMPURE, screenshot-only): read the pure layer, ease the cab attitude, place & draw meshes. No spatial math.
```
Everything left of `render/*` is pure, lives under `src/sim`, contains **no banned token** (`Date|Math.random|performance.now|setTimeout|setInterval`; the noise uses integer hashing, `Math.sin/cos/sqrt` are allowed), and is oracle-tested. The **G3 grep guard glob is widened** to also cover `src/render/quality.ts` (D6'), so the headline "G3 polices the whole spatial model" is literally true.

### 2.1 `src/sim/centerline.ts` (NEW, PURE, TESTED) — the prime seam
```ts
export interface Frame {
  x: number; y: number; z: number;                 // world position of the rail-head centre at chainage s
  tangent: { x: number; y: number; z: number };    // unit forward direction
  up: { x: number; y: number; z: number };          // unit up after cant
  heading: number;                                  // tangent yaw about world-up, rad (= headingAt(s))
  cant: number;                                     // super-elevation roll, rad
}
export function centerlineAt(route: Route, s: number): Frame;
export function heightAt(route: Route, s: number): number;          // formation height = ∫₀ˢ grade (analytic)
export function lateralOffsetAt(route: Route, s: number): number;   // planar offset from datum (analytic)
export function headingAt(route: Route, s: number): number;          // tangent yaw, rad
/** Place a lineside object at chainage s, signed lateral offset d (left −/right +), heading along the track. */
export function placeOnCentreline(route: Route, s: number, d: number): { x: number; y: number; z: number; yaw: number };
/** Camera eye + attitude at chainage s with the driver's LMB look offset (folded from former camera.ts). */
export function cameraFrame(route: Route, s: number, look: { yaw: number; pitch: number }, eye: EyeOffset): CamFrame;
/** PURE target attitude the render layer eases toward; scale=0 ⇒ {0,0} (mobile/reduced-motion). */
export function cabAttitudeTarget(cant: number, grade: number, scale: number): { pitch: number; roll: number };
```
- **Height** (exactly checkable): grades are piecewise-constant `Segment<number>` in `route.grades`. `heightAt(s) = Σ grade_i·(seg overlap with [0,s])`, clamped to `[0,length]`. Piecewise-constant integrand ⇒ piecewise-linear integral ⇒ automatically C0 at every breakpoint. Oracles O1–O3.
- **Planar position** (the curvy one): integrate piecewise-constant `κ`. Heading `ψ(s)=ψ₀+∫κ`. **Per segment, compute the LOCAL arc displacement in the segment's own frame, then ROTATE it by the entry heading `ψ₀` before adding to the running pose** — i.e. for a constant-κ arc of length `L`, `Δlocal = (R(1−cosθ), R sinθ)` with `θ=κL, R=1/κ`, and `Δworld = Rot(ψ₀)·Δlocal`; equivalently `x += (cosψ₀ − cos(ψ₀+θ))/κ`, `z += (sin(ψ₀+θ) − sinψ₀)/κ`. Straight segments advance `L` along `ψ₀`. Accumulate `(x, z, ψ)` at each breakpoint; `ψ` carries across so the tangent is continuous (G1) even though `κ` steps. Oracles O4 (tangent continuity), O5 (single-arc closed form, `ψ₀=0`), **O5b (two-arc compounded pose vs an independent hand-computed reference — catches the rotation-by-ψ₀ that V1's prose omitted)**.
- **cant**: `cant(s) = clamp(CANT_GAIN·|κ(s)|·v_ref(s)², CANT_MAX)`, signed by turn direction, `v_ref = speedLimitAt(s)` (a route property, deterministic). **Pinned: `CANT_GAIN = 0.08`, `CANT_MAX = 0.105 rad (6°)`.** Oracle O6 (incl. a non-tautological point below the clamp + a point above it).
- **`placeOnCentreline(route, s, d)`**: `f = centerlineAt(s)`; world pos `= (f.x + d·rightX, f.y, f.z + d·rightZ)` where `right = (cos f.heading, −sin f.heading)` is the in-plane normal; `yaw = f.heading`. **Every signal head, station, mast, fence, tree, milepost, OLE wire point, person, AWS marker and the rail/sleeper sweep is placed through this** (not `z=chainage,x=±offset`), so the whole lineside follows the curve and faces down the track. Oracle O5c (offset point is exactly `d` from the spine, perpendicular; `d=0` ⇒ on the spine).
- **`cameraFrame` / `cabAttitudeTarget`** (folded from the abandoned `camera.ts` — there is **no** inline trig in today's `scene.ts` to "extract"; these are *new* pure compositions, D1'): `cameraFrame` returns `position = placeOnCentreline(s, EYE_X) lifted by EYE_HEIGHT along up, then clampEye`; `yaw = heading + look.yaw`; `pitch = atan(grade) + look.pitch`; `roll` from `cabAttitudeTarget`. `cabAttitudeTarget(cant, grade, scale) = { pitch: clamp(scale·atan(grade), ±PITCH_MAX), roll: clamp(scale·cant, ±ROLL_MAX) }` — **the ellipsis/duplicate-term in V1 is removed; the formula is exact** (D7, O16). **Pinned: `ROLL_MAX = 0.070 rad (4°)`, `PITCH_MAX = 0.052 rad (3°)`, `attitudeScale = 0.35` (desktop) / `0` (coarse-pointer & reduced-motion).** The **ease** (advancing the live attitude toward the target by `approach()` each frame) is render-local state in `scene.ts`, kept out of `src/sim`.
- `Math.sin/cos/sqrt/atan` allowed; no clock; lives in `src/sim` so G3 guards it. **z-vs-chainage note:** honest planar `z(s) ≤ s` on curves; `physics.ts` integrates speed into the scalar chainage and never reads the 3-D path, so it is unaffected (R6).

### 2.2 `src/sim/terrain.ts` (NEW, PURE, TESTED) — the living land, curvilinear
```ts
export function formationHeight(route: Route, s: number): number;          // = heightAt + ballast shoulder near track
export function naturalGround(route: Route, s: number, d: number): number; // seeded value noise + authored valleys/hills
export function terrainHeight(route: Route, s: number, d: number): number; // blend(formation, natural) by |d|
export function reliefAt(route: Route, s: number): number;                 // natural(s,0) − formation(s)
export function viaductSpanAt(route: Route, s: number): boolean;           // reliefAt < −VIADUCT_THRESH
export function tunnelBoreAt(route: Route, s: number): boolean;            // reliefAt >  TUNNEL_THRESH
export function anchorY(route: Route, s: number, d: number, clearance: number): number; // terrainHeight + clearance
export function clampEye(route: Route, s: number, eyeD: number, eyeY: number): number;  // ≥ terrainHeight(s,eyeD)+MIN_EYE_CLEARANCE
```
**Curvilinear coordinates throughout.** `s` = chainage along the line, `d` = signed lateral offset from the track centre. This is the single fix that makes the blend, the ballast corridor, the formation height, and the feature attribution all **exact on curves** (the V1 `(z, x)` framing was inconsistent — finding #1). The ribbon builder (§2.7) walks `s` and lays verts at offset `d`, so it has `(s, d)` directly — no world→chainage inversion anywhere.

**The blend (the heart):**
```
formation(s)       = heightAt(route, s)                                   // rail top, exact in chainage
natural(s,d)       = baseProfile(route, s, d) + valueNoise(seed, s, d)
w(d)               = smoothstep(0, BLEND_W, |d| − SHOULDER)               // 0 on the track → 1 far out
terrainHeight(s,d) = lerp( formation(s) + ballastShoulder(|d|), natural(s,d), w(d) )
```
**Pinned:** `SHOULDER = 4 m`, `BLEND_W = 80 m`, `BALLAST_DROP = 0.4 m`, `VIADUCT_THRESH = 18 m`, `TUNNEL_THRESH = 12 m`.
- `|d| ≤ SHOULDER` ⇒ `w=0`, terrain **is** formation + ballast shoulder — a clean bed that hugs the *curved* track (O8).
- `|d| ≥ SHOULDER+BLEND_W` ⇒ `w=1`, terrain **is** natural land.
- Between, `smoothstep` handoff: `natural > formation` ⇒ **cutting walls**; `natural < formation` ⇒ **embankment banks**. One function, both reliefs, by the sign of `natural − formation`.

**`naturalGround` (deterministic, banned-token-free):** seeded integer-lattice value noise (mulberry-style hash on `route.terrainSeed + (⌊s/CELL⌋, ⌊d/CELL⌋)`, bilinear, 2–3 octaves) for rolling texture, **plus** authored features keyed off route data in *chainage* space: a **river valley** = a negative Gaussian dip centred on each `route.viaducts[].center`; a **hill** = a positive Gaussian bump centred on each `route.tunnels[].center`. Because both the predicates and the ribbon now evaluate in `(s, d)`, predicate-space and drawn-space coincide by construction (finding #15 dissolves). O9 determinism.

**Predicates** are pure thresholds on `reliefAt` (disjoint, O9). **`anchorY`** = a tested invariant: every prop's base Y = `terrainHeight(s,d) + clearance` (O7). **`clampEye`** floors the eye at `terrainHeight(s, eyeD) + MIN_EYE_CLEARANCE` (`MIN_EYE_CLEARANCE = 0.5 m`) so a pitching camera never ducks under the bed (O20).

### 2.6 Render adapters (IMPURE, screenshot-verified) — no spatial math
- **`src/render/terrain-mesh.ts` (NEW).** Build the **track-following ribbon** ONCE: step `s` at `terrainSegLen`, place the spine via `placeOnCentreline(s, 0)`, lay `terrainSubdiv` lateral verts at offsets `d ∈ [−ribbonHalfWidth, +ribbonHalfWidth]` each at world pos `placeOnCentreline(s, d)` with Y = `terrainHeight(route, s, d)`; normals from finite differences in `(s,d)`. One `BufferGeometry`. To keep it frustum-cullable (finding: a single 14 km mesh is one always-submitted draw call), **split the ribbon into ~16 along-track sections** (each its own bounding box) so off-screen sections cull — still built once, zero per-frame work. PBR `MeshStandardMaterial` + procedural grass/rock/ballast textures, vertexColors grass→rock by slope. A cheap **static far-horizon silhouette band** sits behind it; fog hides the ribbon's far edge.
- **`src/render/scene.ts` (CHG).** (a) Camera/cab read `cameraFrame`; the live cab attitude **eases** toward `cabAttitudeTarget` via `approach()` (render-local). (b) **All lineside placement goes through `placeOnCentreline`**: rails/sleepers swept along the spine; **signal heads, station platforms, OLE masts, fences, mileposts, the contact wire and AWS markers are positioned AND yawed via `placeOnCentreline(sig.chainage, sideOffset)`** (not `z=chainage`), and each **signal lamp/board is oriented to `headingAt(s)`** so it faces back down the real approach tangent (finding #14). (c) **Rain** is anchored to the `cameraFrame` position `(x,y,z)`, not `(0, camera.y, chainage−0.6)` (finding #23). (d) Realism: an **`EffectComposer` chain `RenderPass(linear) → UnrealBloomPass → OutputPass(ACES + sRGB)`** when `bloomEnabled`; **the renderer's own `toneMapping` is set only on the direct-to-canvas (mobile/no-composer) path** so ACES is applied exactly once (finding #11); `toneMappingExposure`/`OutputPass` exposure from `env.exposure`. One **eye-tracking shadow-casting `DirectionalLight`**: a **±100 m ortho frustum that moves with the eye each frame**, `2048²` map (desktop), tuned `bias`/`normalBias`, tight near/far (finding #12), gated by `shadowsEnabled`. A **gradient sky dome** (`material.fog = false`, finding #22) reading the env palette. (e) **Viaduct** (instanced piers to the valley floor + arch spans + river plane) and **tunnel** (portals + dark bore) where the predicates fire. Flat ground deleted.
- **`src/render/scenery.ts` (CHG).** Delete `addBank` + cone-hills (terrain owns relief). Every prop's `(x,z,yaw)` from `placeOnCentreline` and Y from `anchorY` — props follow the curve *and* sit on the terrain; prop detail by `lodForDistance`.
- **`src/render/textures.ts` (NEW).** Canvas-2D tileable grass/rock/ballast/water albedo + normal maps, built once. **No committed image assets.** Wet rails get a cheap **env-tinted specular sheen** (`envMapIntensity` + sky colour), **not** a PMREM reflection (D5').
- **`src/render/cab.ts` (CHG, tiny).** Mount receives eased pitch+roll; furniture unchanged.
- **`src/main.ts` (CHG, tiny).** Use `KINGSGATE_SEAHAVEN`; pass quality opts; project realism params; feed `cameraFrame`/look offset.
- **`scripts/check-dist.mjs` (NEW).** Post-build node script (run via `npm run build && node scripts/check-dist.mjs`): asserts **no `.png/.jpg/.webp/.ktx/.basis` in `dist/`** and `dist/assets/*.js` total **≤ 400 KB gzipped** (O22 — a build-gate, **not** a vitest oracle; finding #3).

## 4. Acceptance criteria (numbered oracles — definition of done)

**Pure unit oracles (vitest, Node, no GPU):**
- **O1 — height integral (analytic).** Single grade `g` over `[0,L]` ⇒ `heightAt(s)===g·clamp(s,0,L)` (≤1e-9); multi-segment equals the hand sum at every breakpoint & midpoint.
- **O2 — height ↔ grade (differential).** Central diff of `heightAt` equals `gradeAt` to 1e-6 off breakpoints.
- **O3 — height C0.** ≤1e-9 across every grade breakpoint.
- **O4 — tangent G1.** Unit tangent continuous (≤1e-6) across every curvature breakpoint; `‖tangent‖≈1`.
- **O5 — single-arc closed form.** First arc (`ψ₀=0`), `θ=κL` ⇒ `x=R(1−cosθ)`, `z=R sinθ` (≤1e-6); straight ⇒ `x` const, `z(s)=s`.
- **O5b — compounded pose (two arcs).** A `straight→arc→arc` (or `arc→arc`) route's accumulated `(x,z,ψ)` matches an independent hand-computed pose (≤1e-6) — **catches the rotate-by-ψ₀ that V1 omitted**.
- **O5c — placeOnCentreline.** `placeOnCentreline(s,d)` is exactly `|d|` from the spine, perpendicular to the tangent; `d=0` ⇒ on the spine; `yaw===heading(s)`.
- **O6 — cant (non-tautological).** 0 on straights; at a specific `(κ,v_ref)` **below** the clamp, `cant` equals the independently computed `CANT_GAIN·|κ|·v_ref²` (gain exercised); at a `(κ,v_ref)` **above**, `cant===CANT_MAX` (clamp bites); sign matches turn; pure.
- **O7 — anchor invariant.** ∀ `(s,d,clearance≥0)`: `anchorY−terrainHeight===clearance` (≤1e-9), `anchorY≥terrainHeight`.
- **O8 — terrain blend (curvilinear).** `|d|≤SHOULDER` ⇒ `terrainHeight===formation+ballastShoulder` (≤1e-9), **exact on curves** because `s` is exact; far out the weight is 1; no NaN/Inf across the route.
- **O9 — predicates disjoint & deterministic.** `viaductSpanAt` true on every authored viaduct chainage, false ≥50 m outside; same `tunnelBoreAt`; never both; `terrainHeight` bit-identical on repeat; changing `terrainSeed` changes the field.
- **O10 — realism monotonicity.** `exposure`/`bloomStrength` finite, ordered (exposure day>dusk>night).
- **O11 — sunDir unit.** `|sunDir|===1` (≤1e-9) ∀ `{time,weather}`.
- **O12 — realism determinism.** New `EnvironmentParams` fields pure; no NaN/Inf across the domain.
- **O13 — quality tier ordering.** coarse-pointer ⇒ `shadowsEnabled=false, bloomEnabled=false, attitudeScale=0`, and `terrainSegLen ≥`, `ribbonHalfWidth ≤`, `terrainSubdiv ≤` desktop.
- **O14 — LOD step (exact + monotone).** `lodForDistance` returns exact `0/1/2` at and just inside `LOD1_DIST/LOD2_DIST` (pinned), and is non-decreasing across the `[0,20000]` sweep.
- **O15 — quality domain sweep.** Every new field finite & in-bounds across `coarsePointer × reducedMotion × {DPR:0,0.5,1,1.5,2,3,4,NaN,Inf}`; `reducedMotion` forces `rainCount=0` & `attitudeScale=0`.
- **O16 — cab attitude clamp (pinned, non-tautological).** `cabAttitudeTarget(0,0,s)==={0,0}`; with `scale=0.35`, a grade/cant **below** the clamp yields exactly `scale·atan(grade)` / `scale·cant`; **above** yields `±PITCH_MAX` / `±ROLL_MAX`; `cabAttitudeTarget(·,·,0)==={0,0}`; roll sign matches cant.
- **O17 — sim/render layering guard (import-scoped).** A test parses the **import specifiers** (not raw text) of `physics.ts`/`simulation.ts` and asserts none resolve to `centerline`/`terrain`/`camera`; ideally a transitive walk over the module graph. The existing **G3** guard (glob widened to `src/sim/*.ts` + `src/render/quality.ts`) passes.
- **O18 — ribbon vertex budget (numeric).** `ribbonVertexCount(coarse,14000) < 25 000` and `ribbonVertexCount(desktop,14000) < 80 000` (pure, from the pinned formula).
- **O19 — route invariants.** `KINGSGATE_SEAHAVEN`: signals ascending; grade/limit/curvature segments tile `[0,length]` with no gap/overlap; stations & every feature band ∈ `[0,length]`; curvature radii ≥ 250 m; viaduct valley depth ≥ 18 m & tunnel hill height ≥ 12 m at centres.
- **O19b — set-pieces on straights.** Every `viaducts/tunnels` band lies on a `κ=0` segment (so chainage ≈ world-z; predicate vs drawn coincide exactly).
- **O20 — eye-clearance clamp.** `clampEye(route,s,eyeD,eyeY) ≥ terrainHeight(s,eyeD)+MIN_EYE_CLEARANCE` ∀ s.

**Build + screenshot gates (definition-of-done for the impure adapters; not vitest oracles):**

## 5. Residual risks
- **R1 — Aesthetic quality isn't machine-checkable.** Oracles prove geometric correctness; only a human confirms it *looks like* real country, **and that the ACES re-tune kept the wet-night reading** (D14). Flagged for Arthur (S1–S4).
- **R2 — Perf on a 14 km route.** Build-once *sectioned* ribbon + tier gates (O13/O18) bound it, but FPS is a sanity check, not an oracle; spot-check desktop + mobile-emulated. The single-mesh always-submitted-draw concern is mitigated by sectioning for frustum culling.
- **R3 — Cab attitude comfort.** O16 bounds it (~4° roll, eased, 0 on mobile); subjective — `attitudeScale`/clamp are tested dials, settable to 0 to fall back to yaw+pitch-only.
- **R4 — `three/examples` under Vite + `base "./"`.** EffectComposer/UnrealBloomPass/OutputPass are ESM sub-imports of installed `three`; if a path won't bundle, fall back to the gradient dome + the direct-to-canvas ACES path and skip bloom (already gated to 0 on mobile) — no new dep.
- **R5 — ACES re-tune effort (NEW).** Switching to ACES de-tunes every emissive landmark; budget an explicit re-balance pass with ENV4 before/after screenshots (D14). Under-estimating this is the top *schedule* risk.
- **R6 — z(s) ≤ chainage on curves.** Eye world-z trails the scalar chainage slightly; harmless (render-only); drive the camera from `centerlineAt(chainage)`, never assume `z===chainage`.
- **R7 — Tunnel interior is cheap** (fog + portal light); upgradeable to a short textured tube (render-only).
- **R8 — Value-noise tuning** must read as gentle rolling land; determinism (O9) guaranteed, pleasantness screenshot-tuned.
- **R9 — Shadow quality on a moving frustum (NEW).** Eye-tracking ortho shadow needs tuned bias/near-far to avoid acne/peter-panning; a render-only tune, screenshot-gated (S4).

## 6. Decision log
- **D1 — The world is math; carve it all pure (curvilinear).** Centreline, terrain, camera are pure functions of route data in `(s, d)` under `src/sim`, oracle-tested, G3-guarded; Three.js is a dumb projector. *(Winner thesis.)*
- **D1' — `camera.ts` folded into `centerline.ts`; drop the "extraction" framing.** There is no inline camera trig in today's `scene.ts` to extract (it sets a fixed eye); `cameraFrame`/`cabAttitudeTarget` are *new* thin compositions, so they live with the prime seam rather than as a separate one-function module. *(Finding #7 — simplifies the module count to two new pure modules.)*
- **D2 — Curvature → planar position via closed-form arcs (rotate by entry heading), not a 2-D plan.** Exact analytic oracles (O5/O5b). A free 2-D bezier plan is the on-ramp-to-nowhere we refuse.
- **D3 — Terrain = blend(formation, natural) by `|d|`, curvilinear; one function for cuttings *and* embankments.** The sign of `natural − formation` does all the work; set-pieces authored via route bands (O8/O9).
- **D4 — Predicates over flags** (`viaductSpanAt`/`tunnelBoreAt`, disjoint O9). No feature flags.
- **D5 — Gradient sky dome baseline** (`fog:false`), reading the env palette; honours the wet-night signature; one mesh.
- **D5' — PMREM deferred (NEW).** A gradient-dome PMREM yields a near-flat, structureless reflection — gold-plating for a cab sim whose dominant reflective surface (wet rails) is glancing. Wet rails get a cheap **env-tinted sheen** instead. Removes a bake/lifecycle/ordering concern from the V2 mental model with no oracle impact. *(Findings #16/#21.)*
- **D6 — Realism params pure; gates live in quality.** environment says what's wanted; quality says what the device allows. No env vars, one device-derived branch.
- **D6' — G3 glob widened to include `src/render/quality.ts`** so "G3 polices the whole spatial model" is literally true. *(Findings #8/#17.)*
- **D7 — Cab rides pitch + clamped, scaled, eased roll** (pure `cabAttitudeTarget`, O16; **pitch = clamp(scale·atan(grade), ±PITCH_MAX)**, no ellipsis; 0 on mobile/reduced-motion; eased in render). Keeps the viaduct bank with a machine-checked, *pinned* comfort bound. *Dissent:* yaw-only is safer; rejected, recoverable by setting `attitudeScale=0`.
- **D8 — Build-once *sectioned* track-following ribbon, NOT chunked streaming.** ~16 along-track sections for frustum culling, built once, zero per-frame work + a far-horizon band + fog. Rejected risk-first's streaming subsystem. Budgeted numerically (O18).
- **D9 — Physics untouched, 1-D, forbidden from importing the 3-D path** (O17, import-scoped).
- **D9' — ENV4 is route-independent.** μ comes from `environmentParams`, not the route; swapping to the Grand Route does **not** move the pin. (V1's worry was misplaced; only the ACES *visual* re-tune needs screenshots — D14.)
- **D10 — Replace the live route, keep `WESTFORD_EASTBANK` as a test fixture** (the signalling/physics tests reference it). New per-route fields optional so it type-checks. *(Finding: deletion would break fixtures.)*
- **D11 — Centreline C0-position / G1-tangent; NO C1 easements.** The eased cab attitude supplies felt smoothness. Rejected VT_RAMP machinery.
- **D12 — Numeric budgets + zero-alloc.** `ribbonVertexCount` (O18, pure) and the post-build asset/size gate (O22, **a build script, not a vitest oracle** — finding #3); zero-alloc is structural (O21 demoted — finding #2).
- **D13 — Eye-clearance clamp + far-horizon band** (`clampEye` O20; static silhouette + fog hide the ribbon edge).
- **D14 — ACES is a palette re-tune, and `test/reach.test.ts` is CHG.** Switching to ACES de-tunes the existing emissive landmarks → budget a re-balance pass with ENV4 before/after screenshots (R5). Adding `QualitySettings` fields breaks `reach.test.ts`'s `toEqual` → update the expected object. *(Findings #13, #14-orig.)*

The ribbon builder walks `s` and lays verts at offset `d`, so it has `(s,d)` directly. *(The headline review fix — findings #1/#10/#24; a net simplification.)*
