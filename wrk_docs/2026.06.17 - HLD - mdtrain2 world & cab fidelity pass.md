# HLD — mdtrain2 world & cab fidelity pass

**Status: SIGNED-OFF** · Approved by Arthur 2026-06-17 · Date 2026-06-17 · Target `C:\language\mdtrain2`
(HEAD `f27f785`, v0.11.0). `package.json` 0.11.0 → **0.12.0**.

Fix 11 reported fidelity defects and produce a north-star **VISION**. Lean process; quality
brief: simplicity, no overengineering, dev sandbox (no migration/back-compat), minimise env
vars, no feature flags, design for testability. Preserve the pure-core (`src/sim/*`,
oracle-tested) / impure-projector (`src/render/*`) split.

Kickoff decisions (Arthur): goal prompt = **north-star vision** (saved as `VISION.md`); curves
= **allow tighter, to ~250 m** (narrow the terrain ribbon); driver = **front-left (UK)**.

**This revision folds the surviving adversarial-review findings** (FEAS-1/3/4/5/6/7/8/9,
TEST-1/2/3/4/5/6/7, SIMP-3/4/6). Killed (refuted): SIMP-1/2/5, FEAS-2.

## 1. Scope
**In:** the 11 defects; a richer/tighter route; denser/varied scenery; `VISION.md`.
**Out (non-goals):** new signalling types (block signals/junctions — aspect core stays
station-starter only); a detailed modelled cab (a clean shell, not a CAD interior); real
motion-streak rain (needs a line/quad system — out of scope; we do *round* drops, the existing
z-slant keeps the motion cue — FEAS-9); day/night cloud / water sim; any new npm dep, env var,
or feature flag.

## 2. The vision (north-star) — to be saved as `VISION.md` on sign-off
> **The Night Cab** is a believable driver's-eye ride of a modern UK electric multiple unit —
> not a game, not a toy. It earns belief two ways: **engineered correctness** (a pure, tested
> simulation-and-geometry core — physics, signalling, the curvilinear world) and **sensory
> fidelity** (a landscape the line genuinely threads through, weather you feel, a cab you sit
> inside). Every addition must make the drive more *believable* or more *beautiful* on a core
> simple enough to hold in your head. We prefer correctness we can test, detail you can see,
> and restraint over feature-sprawl — when in doubt, cut. It runs anywhere a browser does,
> phone to desktop, degrading gracefully.

## 3. Approach (by defect, minimal clean fix)

**A. Driver front-left (#1)** — pure `camera.ts`, comments in `scene.ts`.
- Derived (not guessed): the camera yaws `heading+π` (scene.ts) so camera-local +X → world −X,
  i.e. **screen-right = world −X**. Today `EYE_D=−0.5` puts the eye at world −0.5 = screen-right
  ⇒ driver-RIGHT (mirror-wrong). **Fix: `EYE_D = −0.5 → +0.5`** (eye at world +0.5 = screen-left
  ⇒ driver-left; track vanishing point reads slightly right). **`CAB_SHIFT` stays +0.5** — it is
  in the cabMount-local frame and already puts the pillar to the driver's right; it does NOT
  flip (FEAS-2 refuted the lockstep claim). Flip the stale "driver on the LEFT" comments
  (camera.ts:16, scene.ts:73-75/295) to match.

**B. Cab body (#2)** — impure `cab.ts`.
- Add a **simple enclosing shell** (boxes, built once, parented to the cab mount): left/right
  side walls, roof, floor, rear wall, A-pillars around the existing windscreen surround — dark
  interior, so the world no longer shows around the frame edges. No seat-sign change here (the
  seat is `EYE_D`, §A).

**C. Penalty/DSD discoverability (#3)** — pure selector + `hud.ts` + `help.ts`.
- New **pure** `safetyPrompt(hud) → string | null` (in `controls.ts`, tested): `penalty` ⇒
  `"PENALTY — STOP, THEN PRESS Q"`; `dsdWarning` (and not penalty) ⇒ `"VIGILANCE — PRESS Q"`;
  else `null`. Encodes the real "must be at a stand to clear" in the text.
- `hud.ts` renders that string as ONE prominent centre prompt when non-null, driven by the
  existing `HudView.penalty`/`dsdWarning` — **no new state, no parallel widget** (the DRA/DSD/
  PENALTY lamps stay). `help.ts` row → "Q — Acknowledge AWS / reset penalty (at a stand)".

**D. Tighter, more varied route + narrower ribbon (#4)** — pure `route.ts`, `quality.ts`.
- `quality.ts`: desktop `ribbonHalfWidth` 180 → **120**; coarse 110 → **90**. O-RIBBON floor:
  ribbonHalfWidth ≤ ½·minCurveRadius ⇒ minRadius ≥ **240** (route uses ≥ 250 for margin).
  Note (FEAS-8): ribbon width is **orthogonal to the O18 vertex budget** (ribbonVertexCount
  depends only on terrainSegLen/terrainSubdiv) — narrowing buys **no** perf headroom; it only
  changes lateral extent. R3 draw-call growth is absorbed by instancing/LOD alone.
- `route.ts`: re-author `KINGSGATE_SEAHAVEN` curvature: **≥ 5 non-zero curvature segments**,
  including **≥ 1 S-bend** (adjacent +κ then −κ) and **≥ 1 tight curve R < 400** (down to ~250).
  **Bands stay wholly on κ=0 segments** (viaduct 7730-8370, tunnel 11320-12080 — O19b). Author a
  **PSR** (lower speedLimits) on the curves such that the *unclamped* cant at the posted speed
  stays within the cap: `CANT_GAIN·|κ|·speedLimit² ≤ CANT_MAX` per curve (TEST-4 — keeps each
  bend off the permanent 6° ceiling; no cant-formula change). `WESTFORD_EASTBANK` unchanged.

**E. OLE: stagger + connection + richer masts (#6, #7, #8)** — impure `scene.ts` (one mast grid).
- **One span unit = mast-to-mast (45 m).** Drive everything off the existing mast loop (index
  `i`, `side = i%2 ? −1 : +1`). The contact-wire lateral offset at mast `i` is
  `stagger_i = side · STAGGER` (≈0.2 m) — **inline**, no separate pure module/AC (SIMP-3; and
  it must not live in Three-importing `scene.ts` as a "pure" export tested from `controls.ts`
  — TEST-3). Rebuild `buildContactWire` to sweep **mast-to-mast through the staggered points**
  (segment from `(s_i, stagger_i)` to `(s_{i+1}, stagger_{i+1})`) ⇒ visible zigzag.
- Richer **mast**: keep masts/arms as `InstancedMesh`; add the **registration arm + dropper** as
  a **third `InstancedMesh`** whose per-instance transform reaches from the mast to that mast's
  staggered wire point — so the wire **connects** to the support. **No per-mast `Group`** (keeps
  instancing / the R3 budget — SIMP-6). Masts alternate sides as today.

**F. Round raindrops (#9)** — `textures.ts` + `scene.ts`.
- Add a soft **round** alpha texture (CanvasTexture radial gradient; `makeGlowTexture` is the
  precedent) on the rain `PointsMaterial.map` (transparent, depthWrite false). **Round only** —
  Points sprites are screen-axis-aligned squares and cannot be oriented to the motion vector, so
  no true streak; the existing z-slant scroll keeps the motion cue (FEAS-9).

**G. Visible signals (#10)** — impure `scene.ts` `buildSignals` (+ anchor with §H).
- Make the head **read**: lighter grey post + a proper backboard, a larger lit aspect, a faint
  always-on lamp housing so the head is legible even unlit. Count stays 3 starters — no
  signalling-core change (DL4).

**H. Seat stations & signals on the ground (#11, #10-float)** — `scene.ts` + `scenery.ts`.
- **Corrected diagnosis (FEAS-3):** the float is a **uniform 0.40 m lip** — at the station
  platform offset (~2.4 m) and signal offset (~2.9 m), both inside SHOULDER+BLEND_W (84 m) of the
  spine, `terrainHeight` is the **near-track ballast bed = formationHeight − 0.4** at *every*
  site (incl. the deep Brinemouth/Seahaven formations). It is **not** a terrain-follow over
  cuttings. **Fix: anchor the structure base to `terrainHeight(route, s, offset)`** (= bed) rather
  than `formationHeight`, dropping it the uniform 0.4 m so the slab/post meet the ballast.
- **Dependency (FEAS-4):** `buildPlatformPeople` (scenery.ts) anchors people to
  `formationHeight+platTop` to match the slab top. If the slab base moves, **move the people by
  the identical amount** — route both off **one shared offset** (the slab top stays at the same
  world height; only the slab *base*/footing drops, so the people may not need to move if the
  slab TOP is unchanged — confirm in implement and keep them on one constant either way).

**I. Distant edge treatment (#5 edge) — CHEAPEST-FIRST, conditional (SIMP-4, FEAS-6).**
- Narrowing the ribbon to 120 m brings the ground edge closer. **Do NOT pre-build an edge belt.**
  Order: (1) narrow + screenshot; (2) if the edge reads hard, pull `env.fogFar` / the ribbon
  fade in (a constant change, no geometry); (3) **only if still hard**, add a distant belt — and
  if so it MUST: skip `viaductSpanAt` ±SHOULDER_RAMP(120 m) margin, **cap element height** so it
  cannot occlude the viaduct valley from eye height, use **instanced trees** (sink to base) not
  "low hills" near bands, and pass a **REG-vis** screenshot (viaduct valley still visible) — i.e.
  not repeat the removed far-band occlusion bug.

**J. Denser, varied scenery (#5 density)** — `scenery.ts`.
- More **trees** (denser), new **bushes/hedgerows** (low instanced foliage), more **buildings**
  (a city cluster ~Kingsgate 0-1800; suburban ~Ashcombe), 1-2 more **bridges**. All instanced,
  placed via `placeOnCentreline`+`anchorY`, **skipping `viaductSpanAt`/`boreCorridorAt`**,
  density falling off with |d|. LOD-aware. (No new placement subsystem — reuse the existing
  instanced builders.)
